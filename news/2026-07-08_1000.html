C-03Automating AI Away: How Deterministic Tools Should Sandwich Non-Deterministic LLMs

📰 Article (English)

A Hacker News front-page post from Replicated's engineering blog argues that the key to reliable AI-assisted development is not to expect perfect LLM behavior but to architect deterministic tools around the non-deterministic model. The author's core thesis: LLMs like Claude are simultaneously brilliant at code analysis and frustratingly inconsistent — the model committed build artifacts into git twice despite explicit instructions not to, and would attempt manual parsing "torturously" even when told not to. The proposed solution is to "sandwich that brilliant but inconsistent non-determinism between powerful deterministic tools." Concretely, this means: provide fast, reliable tools (fast feedback loops), build formal workflows that constrain what the model can do, and create self-correcting mechanisms. As the LLM handles repeated action sequences, those patterns become automated into deterministic tools — effectively letting the LLM automate itself away. The technical implementation (Beagle SCM) enables LLMs to "script their own routines" in JavaScript, with heavy computation in C and tool/workflow layers in JavaScript. The pattern generalizes beyond this specific tool: define clear tool contracts, keep the LLM in the orchestration layer, push deterministic work into the tool layer. This is a practical agentic engineering post grounded in production experience.

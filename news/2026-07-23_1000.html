B1⭐ 仕様駆動開発で「仕様とコードのズレ」をハッシュで決定的に検出するツールを作った

📰 本文 + 和越対訳

AIを活用した仕様駆動開発（spec-driven development）において、仕様書とコードが乖離していく問題を解決するためのツールが開発された。このツール「artgraph」は、仕様ファイルと実装ファイルの内容からハッシュ値を生成し、両者の整合性をCI/CDパイプライン上で自動的に検出する仕組みを提供する。AIがコードを自動生成する時代において、「仕様を書いてAIに実装させる」というワークフローが普及しているが、仕様が更新されてもコードが追随していない、あるいはコードが変更されても仕様が更新されないという問題が頻発している。artgraphはこのズレを決定論的（deterministic）に検出し、開発チームが気づかないまま仕様と実装が乖離するリスクを大幅に低減する。Pull Requestのレビュー段階でハッシュ不一致の警告が出ることで、仕様とコードの同期が強制される。

Một công cụ mới được phát triển để giải quyết vấn đề "spec và code bị lệch nhau" trong quy trình phát triển hướng đặc tả (spec-driven development) sử dụng AI. Công cụ "artgraph" tạo ra giá trị hash từ nội dung file đặc tả và file implementation, sau đó tự động kiểm tra tính nhất quán của hai bên trên pipeline CI/CD. Trong thời đại AI tự động sinh code, quy trình "viết spec rồi để AI implement" ngày càng phổ biến nhưng kéo theo rủi ro spec và code bị lệch nhau mà không ai hay biết. artgraph phát hiện sự lệch lạc này một cách xác định (deterministic), buộc đồng bộ hóa spec và code tại giai đoạn review pull request.

The project reached the top of Hacker News with 629 points, reflecting strong developer interest in "local-first" and "offline-capable" tooling that runs entirely in the browser. The system supports collaborative editing through peer-to-peer synchronization, data binding for charts and dynamic content, and all of this operates without touching a backend server. This architecture represents a new category of web tooling where the delivery medium (HTML) and the application itself are the same artifact, eliminating the traditionally layered build-and-serve stack. For developers building internal tools or demos that need to survive without internet access, Bento offers a compelling model.

B-01Claude Code Artifacts が MCP コネクタ対応——スナップショットから「生きたダッシュボード」へ

📰 本文 + 和越対訳

Claude Code の Artifacts 機能が MCP（Model Context Protocol）コネクタと連携できるようになり、publish した HTML ページがリアルタイムで外部データを取得できるようになった。これまでの Artifacts は publish 時点のデータが固定されており、翌日には内容が古くなるという根本的な制約があった。今回の対応により、ページが開かれるたびに GitHub コネクタや Google Calendar コネクタなどを通じて最新データを自動取得し、「生きたダッシュボード」として機能させることが可能になった。重要な設計上の点として、コネクタ呼び出しは publish した人のアカウントではなく、ページを閲覧している人のアカウントで実行される。この設計により、組織内で共有した場合でも各メンバーが自分の権限の範囲内でデータを参照するため、認証情報が漏れることなくセキュアな共有ができる。利用には Claude Code v2.1.209 以降が必要で、ローカル MCP サーバーではなく claude.ai 側のコネクタのみが対応している点に注意が必要だ。

Tính năng Artifacts của Claude Code hiện đã tích hợp với MCP (Model Context Protocol) connector, cho phép các trang HTML được publish có thể lấy dữ liệu thực tế mỗi lần được mở. Trước đây, Artifacts chỉ cố định dữ liệu tại thời điểm publish và trở nên lỗi thời vào ngày hôm sau. Với tính năng mới này, mỗi khi trang được mở, GitHub connector, Google Calendar connector, v.v. sẽ tự động lấy dữ liệu mới nhất, biến Artifacts thành "dashboard sống". Một điểm thiết kế quan trọng là connector được gọi bằng tài khoản của người xem, không phải người publish — đảm bảo bảo mật khi chia sẻ trong tổ chức. Cần Claude Code v2.1.209 trở lên và chỉ hỗ trợ claude.ai connector (không hỗ trợ MCP server local).

📰 本文 + 和越対訳

Claude Code、Codex、GitHub Copilot のような AI コーディングエージェントはもはや開発者の PC だけで動く存在ではなくなっている。git worktree を使った分離、一時的なクラウドコンテナ、GitHub Actions の ephemeral environment など、実行場所は分散している。この現実を前に「全エージェントの作業をローカルの1台で監視する」という前提が崩れており、実行中の途中状態をローカルで把握することは難しい。しかし変更が main へ入る境界は共有できる——それが GitHub 上の branch と PR だ。この記事では、AIエージェント時代における main 保護の考え方を整理し、sandbox や worktree による実行分離と、PR レビュー・merge policy による境界制御を明確に分けて論じている。コントロールポイントをエージェントの内部ではなく GitHub 側に置くことで、vendor に依存しない統合判断の場所を確保できるという主張だ。

Các AI coding agent như Claude Code, Codex, GitHub Copilot hiện không còn chỉ chạy trên máy tính của developer. Chúng hoạt động trong git worktree, cloud container tạm thời, ephemeral environment của GitHub Actions — và việc giám sát tất cả bằng một máy local đã không còn khả thi. Tuy nhiên, ranh giới mà mọi thay đổi phải đi qua trước khi vào main thì vẫn có thể kiểm soát chung — đó chính là branch và PR trên GitHub. Bài viết phân tích rõ sự khác biệt giữa việc bảo vệ quá trình chạy (sandbox, worktree) và bảo vệ điểm đến (PR review, merge policy). Đặt điểm kiểm soát ở phía GitHub, không phải trong từng agent, giúp có một mặt phẳng điều phối không phụ thuộc vendor.

C-03TP-Link Kasa Cameras Leaked Home GPS Coordinates via Unauthenticated UDP for 6 Years

📰 Article (English)

A security researcher disclosed that TP-Link Kasa smart cameras (model EC71) had been broadcasting the physical GPS coordinates of users' homes over an unauthenticated UDP port for approximately six years. Any device on the same network — or, in some configurations, from the open internet — could query the camera and receive the precise latitude/longitude of the installation address. The vulnerability required no authentication, no credentials, and no prior access to the user's TP-Link account. The researcher discovered the issue during routine IoT research and responsibly disclosed it to TP-Link. The flaw appears to stem from a poorly designed local discovery protocol that included geolocation data as part of the device broadcast payload. TP-Link has since released a firmware patch. The incident highlights the persistent problem of IoT devices shipping with overly permissive local network APIs, where internal debug/discovery endpoints are left exposed even in consumer products deployed for years without security audits.

🟡 LOW If you own a Kasa EC71 camera, update firmware now. More broadly, audit IoT devices on your home/office network for exposed UDP ports.

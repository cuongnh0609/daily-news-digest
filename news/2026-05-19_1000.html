B-01Claude Rulesが無視される本当の理由——/btw・/rewind・/clearでコンテキストを立て直す

📰 本文 + 和越対訳

Claude Codeで`.claude/rules`に定義したルールが会話の後半になるにつれて無視されるようになる原因は、コンテキストウィンドウの劣化にある。会話開始時はルールが数百トークンを占めていても、エラー修正のやり取りが積み重なると全体が数千トークンに膨らみ、相対的にルールの比重が薄れる。この問題を解決するために効果的なコマンドが3つある。まず`/btw`は、メインのコンテキストを汚染せずに方向確認ができるサイドバーを開く。次に`/rewind`は、過去のチェックポイントまで会話を巻き戻し、蓄積したエラーを一掃する。そして`/clear`はコンテキスト全体をリセットし、コードの変更内容はそのまま残しつつ、まっさらな状態で次のタスクに取り掛かれる。また`/branch`で会話履歴を分岐させたり、`claude -w`でgit worktreeを使った物理的な作業ディレクトリの分離も有効だ。Claude APIはステートレスであり、Claude Codeは毎回全会話履歴を送信する仕組みのため、こうした能動的なコンテキスト管理が出力品質の維持に直結する。

Khi dùng Claude Code, các rule được định nghĩa trong `.claude/rules` thường bị bỏ qua dần khi hội thoại kéo dài — nguyên nhân là "context window degradation": phần rule ban đầu chỉ chiếm vài trăm token nhưng bị chìm lấp khi trao đổi sửa lỗi tích lũy lên hàng nghìn token. Có 3 lệnh để khắc phục: `/btw` mở sidebar không làm ô nhiễm context chính để kiểm tra hướng đi; `/rewind` quay về checkpoint cũ, xóa sạch các lỗi tích lũy; `/clear` reset toàn bộ context trong khi vẫn giữ các thay đổi code. Ngoài ra, `/branch` cho phép rẽ nhánh lịch sử hội thoại, và `claude -w` dùng git worktree để hoàn toàn tách biệt môi trường làm việc song song. Vì Claude API là stateless và Claude Code gửi lại toàn bộ lịch sử mỗi lần, việc chủ động quản lý context là chìa khóa duy trì chất lượng output.

B3Claude Codeの指示追従性を高める3つの仕組み——スラッシュコマンド・スキル・フックの使い分け

📰 本文 + 和越対訳

「CLAUDE.mdにルールを書いても守られないことがある」という問題を解決するための、3つの仕組みの使い分けを解説した記事が公開された。スラッシュコマンドは「流れ」を定義する仕組みで、.claude/commands/ディレクトリに置いたMarkdownファイルが/コマンド名で呼び出せる。スキルは「方法」を定義するもので、Claudeがコンテキストに応じて必要な時だけ読み込む。そして最も強制力が高いのがHooksで、特定のイベント（ツール呼び出し前後など）でシェルスクリプトを実行し、exit code 2を返すことでツールの実行を強制停止できる。著者はZennへの記事投稿ワークフローに実際に適用した経験を紹介しており、「pushの前に必ず確認する」というルールをPreToolUseフックで強制することで、Claudeが誤ってgit pushを実行してしまうリスクを構造的に排除した。「Claudeは忘れるかもしれないが、シェルは絶対に忘れない」という言葉が印象的だ。CLAUDE.mdは常にコンテキストにあるが、スキルは必要時のみ読み込まれるためコンテキストの節約にもなる。

Một bài viết đã giải thích cách phân biệt và kết hợp ba cơ chế để nâng cao độ tuân thủ chỉ thị trong Claude Code. Slash commands định nghĩa "luồng công việc" — file Markdown trong thư mục .claude/commands/ được gọi bằng /tên-lệnh. Skills định nghĩa "cách thực hiện" và chỉ được Claude nạp vào khi cần thiết, giúp tiết kiệm context. Hooks là cơ chế mạnh nhất: chạy shell script tại các lifecycle event (trước/sau khi gọi tool) và có thể buộc dừng hoàn toàn bằng cách trả về exit code 2. Tác giả áp dụng thực tế vào workflow đăng bài Zenn, dùng PreToolUse hook để bắt buộc xác nhận người dùng trước khi git push — loại bỏ hoàn toàn rủi ro Claude vô tình push code. Câu nói đáng nhớ: "Claude có thể quên, nhưng shell thì không bao giờ quên." Đây là bài giới thiệu thực chiến cho cuốn sách "実践Claude Code入門."

C1ProgramBench: Can Language Models Rebuild Complex Programs From Scratch? Best Models Pass Only 3% of Tasks

📰 Article (English)

Researchers from Stanford, Meta, and collaborating institutions have published ProgramBench, a new benchmark designed to evaluate whether software engineering agents can develop complete programs from scratch based on documentation alone. Unlike existing benchmarks that focus on narrow bug-fixing or snippet-completion tasks, ProgramBench measures holistic development capability across 200 tasks spanning from simple command-line utilities to complex software systems including FFmpeg, SQLite, and PHP. The results are sobering: none of the nine evaluated language models fully resolved any task. The best-performing model passed 95% of tests on only 3% of tasks. Models consistently produced monolithic, single-file implementations rather than architecturally structured codebases resembling human-authored software. The evaluation methodology uses end-to-end behavioral testing generated through agent-driven fuzzing, avoiding prescriptive implementation requirements that would unfairly constrain approaches. The benchmark highlights a significant gap between narrow coding benchmarks (where LLMs appear increasingly capable) and the full software engineering lifecycle. Building real software requires architectural decisions, multi-file organization, documentation-driven design, and consistent naming conventions that current models struggle to maintain across a large generation task. The paper is authored by John Yang, Kilian Lieret, and colleagues, submitted to arXiv on May 5, 2026.

C2SQLite Officially Recognized as Library of Congress Recommended Storage Format

📰 Article (English)

The United States Library of Congress has officially designated SQLite as a Recommended Storage Format for digital datasets, placing it alongside XML, JSON, and CSV as formats considered optimal for long-term preservation and accessibility of digital materials. This recognition carries significant weight: the Library of Congress evaluates formats on seven rigorous criteria — Disclosure (comprehensive public specifications), Adoption (widespread current use), Transparency (direct human-readable inspection), Self-documentation (built-in metadata), External Dependencies (minimal platform requirements), Patent Impact (no royalty restrictions), and Technical Protection Mechanisms (no encryption barriers to archival access). SQLite scores well across all dimensions: its file format specification is fully public and extensively documented, the format is among the most widely deployed in the world (billions of devices), a plain file can be opened and partially read with standard tools, and the project is explicitly placed in the public domain with no patent claims. The designation reinforces SQLite's position not just as a development database but as an archival-grade data format — a notable shift in how the broader technology community might approach SQLite for long-term data persistence use cases, edge computing, and offline-first application design.

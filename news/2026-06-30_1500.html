06Claude Code の Agent Teams で「PM＋作業者」AIチームを構築——複数課題の並列処理と自己改善を実装

📰 本文 + 和越対訳

AccentureのエンジニアがZennに公開した記事では、Claude Codeの実験的機能「Agent Teams」を使って、PMエージェント1体と複数の作業者エージェントで構成されるAIチームを構築した実践が詳しく紹介されている。Agent Teamsとは、リードエージェントがteammate（チームメンバーエージェント）をspawn（生成）して並列作業させ、メールボックス経由でエージェント間が連絡を取り合える機能で、Claude Code v2.1.32以降で利用可能だ（要：環境変数CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1）。この構成では、PM役エージェントがタスクを課題に分解してteammateに割り振り、各作業者は独立して並列で動作したあとPMに報告する。PMはその報告を独自に裏取り検証してから「確定」とする設計で、品質担保とスピードを両立させている。著者によれば、「丸投げ」でも「全部手動確認」でもなく、自走と検知・引き戻しの構造を持つ設計が重要だとのことだ。本記事は2026年6月時点のv2.1.186系での挙動をもとにしており、ベータゆえに仕様変更の可能性がある。

Một kỹ sư từ Accenture Japan đã công bố trên Zenn bài viết chi tiết về việc xây dựng đội AI gồm 1 agent đóng vai PM và nhiều agent "nhân viên" sử dụng tính năng thực nghiệm Agent Teams của Claude Code. Agent Teams cho phép agent dẫn đầu spawn (tạo ra) các teammate chạy song song và giao tiếp qua mailbox, có thể dùng từ Claude Code v2.1.32 trở lên (cần bật biến môi trường CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1). Trong cấu hình này, PM phân rã task thành các vấn đề nhỏ, phân công cho teammate xử lý độc lập song song, sau đó mỗi agent báo cáo về PM. PM sẽ độc lập xác minh kết quả trước khi "chốt" — thiết kế nhằm đảm bảo cả chất lượng lẫn tốc độ. Tác giả nhấn mạnh không nên "khoán trắng" hay "kiểm tra tay hoàn toàn" mà cần thiết kế cấu trúc có khả năng tự chạy, phát hiện sai lệch và kéo lại đúng hướng.

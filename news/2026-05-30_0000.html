B1MCPでAgentの能力を増やすのをやめたら「Context Boundary」が見えてきた——LLMシステム設計の新視点

📰 本文 + 和越対訳

Model Context Protocol（MCP）をただのAgent能力拡張ツールとして扱うのをやめると、より本質的な設計原則が見えてくる——そんな知見を共有した技術記事がZennに投稿された。著者は、LLMが「何でも答えられる」という特性がむしろリスクになることを指摘する。特に契約書レビューや法律判断などの専門ドメインでは、RAGから取得された複数の論点が自然言語の中で平均化・混在してしまい、重要な矛盾点や責任の所在が曖昧になりやすい。この問題を解決するため、著者はMCPをscope check（回答対象の確認）、candidate review lenses（観点提示）、output boundary（出力制限）という3つの境界として設計するアプローチを提案する。重要なのは「事後の免責文ではなく設計段階での制約」であり、これによってAgentが不適切な観点で回答することを構造的に防ぐことができる。「No disclaimer by design」という思想は、本番運用のAIエージェント設計における新しい指針となりえる。

Một bài viết kỹ thuật trên Zenn chia sẻ góc nhìn mới về thiết kế MCP (Model Context Protocol): thay vì chỉ dùng MCP để mở rộng khả năng của Agent, hãy dùng nó như một cơ chế kiểm soát ngữ cảnh. Tác giả chỉ ra rằng đặc tính "có thể trả lời mọi thứ" của LLM thực ra là rủi ro trong các domain chuyên biệt như review hợp đồng — khi RAG trộn lẫn nhiều luận điểm, các mâu thuẫn quan trọng dễ bị san phẳng. Giải pháp đề xuất là thiết kế MCP với ba lớp ranh giới: scope check (xác nhận phạm vi trả lời), candidate review lenses (trình bày quan điểm), và output boundary (hạn chế đầu ra). Triết lý "No disclaimer by design" — ràng buộc ngay từ khâu thiết kế thay vì disclaimer hậu kỳ — là hướng dẫn hữu ích cho việc triển khai AI agent trong môi trường production.

📰 Article (English)

Kog AI unveiled the Kog Inference Engine (KIE), achieving 3,000 output tokens per second on eight AMD MI300X GPUs and 2,100 tokens per second on an equivalent NVIDIA H200 cluster — at batch size 1, meaning single-request real-time inference, not batched throughput. The company argues this demonstrates that the inference speed bottleneck is a software problem, not a hardware one. KIE treats decoding as a memory-bandwidth maximization problem rather than a raw FLOPS problem. Key engineering choices include a persistent monokernel design that eliminates costly kernel boundary crossings and CPU scheduling overhead, a custom communication library called KCCL that reduces all-reduce latency to under 3 microseconds, and a Delayed Tensor Parallelism architecture that overlaps communication with computation instead of blocking on it. The system currently operates a 2B parameter model, with scaling projections suggesting 1,000–5,000 tokens/s for frontier mixture-of-experts models on next-generation GPUs. Kog specifically calls out agentic workflows as the target use case — AI agents running reasoning loops need single-request decoding speed far more than they need high-batch throughput. The practical implication: faster agents can react, plan, and iterate in real time without the latency that currently makes long-horizon agent tasks feel sluggish.

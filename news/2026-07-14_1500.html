06AIがホームディレクトリを丸ごと削除——Docker が明かすコーディングエージェントの構造的欠陥と sandbox 解決策

📰 本文 + 和越対訳

Claude Code、Cursor、Replit、Kiroなど主要なAIコーディングエージェントが、ユーザーのホームディレクトリを誤って削除するという深刻な事故が相次いでいることをDockerが公式ブログで報告した。2025年12月にはRedditユーザーが「Claude Codeにリポジトリの整理を依頼したところ、15年分の写真を含むホームディレクトリ全体が消えた」と報告。Dockerの分析によれば、エージェントが「プロジェクトフォルダを削除する」と予測してコマンドを生成した際、シェルが ~ をユーザーのホームディレクトリに展開してしまうという根本的なミスマッチが原因だ。同様のインシデントはUbuntu/WSL2環境でも複数確認されており、エージェントが計画段階と実行段階で異なる解釈をすることへの対処が急務となっている。Dockerはこの問題に対し、マイクロVM技術を用いた「Docker Sandboxes」を発表。エージェントのアクセス範囲をプロジェクトディレクトリのみに限定し、システム全体への破壊的操作を物理的に防ぐアーキテクチャを提示した。

Docker đã công bố trên blog chính thức về một loạt sự cố nghiêm trọng khi các AI coding agent như Claude Code, Cursor, Replit và Kiro vô tình xóa toàn bộ thư mục home của người dùng. Tháng 12/2025, một người dùng Reddit báo cáo "sau khi nhờ Claude Code dọn dẹp repository, toàn bộ thư mục home bao gồm 15 năm ảnh đã biến mất". Phân tích của Docker chỉ ra nguyên nhân cốt lõi: khi agent tạo lệnh "xóa thư mục project", shell mở rộng ký tự `~` thành thư mục home của người dùng thay vì thư mục project như agent dự đoán — một sự không khớp cơ bản giữa giai đoạn lập kế hoạch và thực thi. Nhiều sự cố tương tự đã được xác nhận trên môi trường Ubuntu/WSL2. Để giải quyết, Docker giới thiệu "Docker Sandboxes" sử dụng công nghệ microVM, giới hạn phạm vi truy cập của agent chỉ trong thư mục project, về mặt kiến trúc ngăn chặn các thao tác hủy hoại trên toàn hệ thống.

10Jacquard: A Programming Language Designed for AI-Written, Human-Reviewed Code — Effects, Uncertainty, and Content-Addressable Identity

📰 Article (English)

As AI systems write an increasing proportion of production code, a new programming language called Jacquard proposes that programming languages themselves must evolve to serve a world where machines write and humans review. Published by FriendMachine as a research project, Jacquard makes three properties that reviewers need explicit at the language level rather than requiring full code comprehension. First, effect signatures: every function declares exactly what capabilities it may use — network access, file system, database, etc. — as part of its type signature. A function typed as (text) ->{net} text can only touch the network; it cannot silently read your disk. Second, uncertainty handling: programs can express finite discrete probability models and enumerate all possible outcomes with exact probabilities, making the "how sure are we" question answerable without manual tracing. Third, content-addressed identity: code is hashed by its canonical structure rather than source bytes, so comments and formatting changes do not trigger unnecessary reruns — the hash answers whether two versions are semantically equivalent. Jacquard compiles natively to C with byte-identical output, includes a test framework called Warp, and ships a standard library written in Jacquard itself. The language is explicitly designed for AI agents to read and write while humans audit — a direct response to the "spec-driven development" paradigm emerging in agentic engineering.

11Simon Willison: Cache uvx Tools in GitHub Actions With a Date-Pinned UV_EXCLUDE_NEWER Trick

📰 Article (English)

Simon Willison published a practical tip today for Python developers running tools via uvx inside GitHub Actions workflows: use the UV_EXCLUDE_NEWER environment variable to pin tool resolution to a specific date, then include that date in the GitHub Actions cache key. This seemingly simple change solves a common frustration — repeated PyPI network calls on every CI run to download the same tool versions. The mechanism works because UV_EXCLUDE_NEWER: "2026-07-12" tells uv to only consider package versions published before that date, making dependency resolution deterministic. When paired with GitHub Actions' standard actions/cache step using the date as part of the cache key, the tools are downloaded once, cached, and reused on subsequent runs. To upgrade tools, simply update the date — the cache key changes, triggering a fresh download of the newest versions available up to the new date. This pattern brings two benefits that complement each other: reproducibility (same date = same tools, every time) and intentional control over upgrades (no surprise breaking changes from auto-updating dependencies). Willison notes this is particularly valuable for teams running many parallel CI jobs that would otherwise each spend time fetching identical tool sets. The broader observation is that uv's date-based version resolution unlocks caching strategies that weren't possible with older Python packaging tools like pip or pipx.

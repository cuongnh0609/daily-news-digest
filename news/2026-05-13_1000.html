C1LangChain Delta Channels: Solving the O(N²) Storage Problem for Long-Running Agents

📰 Article (English)

LangChain's engineering team published a major runtime update for Deep Agents called Delta Channels, which solves a fundamental scalability issue in long-running agent workloads. The LangGraph runtime historically saves a full snapshot of agent state at every step — a design that enables time-travel debugging and human-in-the-loop interrupts, but causes storage to grow at O(N²). A coding agent running 200 turns would produce approximately 5.3 GB of checkpoint data under the old model. Delta Channels changes the storage model: instead of full snapshots, only incremental diffs (deltas) are stored at each step. Complete snapshots are still written periodically (every K steps), allowing reconstruction from any checkpoint. Under this scheme, the same 200-turn agent produces just 129 MB — a 41× reduction. The migration is transparent: existing agent threads and all LangGraph features (interrupts, time-travel, human-in-the-loop) continue working without any data migration. This update ships in LangGraph Cloud and will be available for self-hosted LangGraph Server in a future release. For teams running production agents with extended context windows and tool-heavy workflows, this change eliminates a hidden infrastructure cost that grows quadratically with agent complexity.

Delta Channels thay đổi mô hình lưu trữ: thay vì full snapshot, chỉ lưu các diff (delta) tăng dần ở mỗi bước. Snapshot đầy đủ vẫn được ghi định kỳ (mỗi K bước) để có thể khôi phục từ bất kỳ checkpoint nào. Với mô hình này, cùng agent 200 lượt chỉ cần 129 MB — giảm 41 lần. Migration hoàn toàn trong suốt: các agent thread hiện có và tất cả tính năng LangGraph (interrupts, time-travel, human-in-the-loop) tiếp tục hoạt động không cần di chuyển dữ liệu. Bản cập nhật này đã có trong LangGraph Cloud và sẽ có trong LangGraph Server tự host ở phiên bản tương lai. Với các team đang chạy production agent với context window mở rộng và workflow nặng về tool-use, thay đổi này loại bỏ một chi phí infrastructure ẩn tăng theo bình phương độ phức tạp của agent.
Nếu bạn đang tự host LangGraph Server hoặc dùng LangGraph Cloud cho production agents, hãy plan migrate sang Delta Channels — storage cost sẽ giảm đột ngột khi agent complexity tăng. Đây là breaking change về infra cost model, không phải chỉ optimization.

C2DeepMind: Reimagining the Mouse Pointer for the AI Era — Context-Aware Cursor That Understands Intent

📰 Article (English)

Google DeepMind published research on rethinking the mouse pointer — a UI primitive that has remained essentially unchanged for fifty years — for an age where AI can understand visual context and user intent. The core problem they identify: AI tools today exist in separate windows or sidebars, forcing users to break their workflow and copy content into dedicated AI applications. The DeepMind vision is an "AI-native pointer" that meets users inside any application without disrupting their flow. The research outlines four principles for this reimagined interaction model. First, continuity — the AI maintains context across all applications the user switches between. Second, automatic visual capture — the system reads visual context (images, charts, UI elements) without requiring the user to explicitly describe them. Third, natural language at the point of action — commands like "fix this," "summarize that," or "schedule this date" issued directly at the cursor position. Fourth, structured data extraction — visual elements like dates, names, and locations are automatically parsed into structured, actionable entities. While still in research phase, this work signals a broader DeepMind push into ambient computing — AI that operates within the OS layer rather than as a discrete application.

C3GitHub Copilot Code Review Gets Severity Levels and Comment Grouping — Shipped to All Users

📰 Article (English)

GitHub shipped two significant improvements to Copilot Code Review, now available to all users opted into the new pull requests experience. The changes address the two biggest friction points in AI-assisted code review: noise and prioritization. The first change is severity labeling. Every Copilot code review comment now carries a severity level: High, Medium, or Low, displayed in the top-right corner of the comment. This lets developers triage AI suggestions the same way they would triage bug reports — address High severity items first, defer Low severity ones. Previously, all suggestions appeared with equal visual weight, making it difficult to decide what to fix before merging. The second change is comment grouping. Copilot now groups similar suggestions across a pull request into a single comment instead of repeating the same feedback for every occurrence. For example, if Copilot identifies a variable naming issue present in multiple places, it raises the concern once and references all affected locations. This is particularly impactful on large PRs where AI review noise was previously amplified. Both features ship alongside an updated suggested changeset UI that makes applying and dismissing suggestions faster. This is part of GitHub's broader effort to make Copilot reviews feel more like a senior engineer's feedback rather than a linter.

Các thay đổi nhắm vào hai điểm ma sát lớn nhất trong code review được hỗ trợ bởi AI: nhiễu và ưu tiên hóa. Thay đổi đầu tiên là nhãn mức độ nghiêm trọng (severity labeling). Mỗi comment của Copilot Code Review giờ có mức độ: High, Medium hoặc Low, hiển thị ở góc phải trên của comment. Điều này cho phép developer triage các gợi ý AI như triage bug report — xử lý High trước, để Low sau. Trước đây tất cả gợi ý đều có trọng số hiển thị bằng nhau, khiến khó quyết định cần fix gì trước khi merge. Thay đổi thứ hai là nhóm comment. Copilot giờ gom các gợi ý tương tự trong pull request thành một comment duy nhất thay vì lặp lại cùng phản hồi cho mỗi lần xuất hiện. Ví dụ, nếu Copilot phát hiện vấn đề đặt tên biến ở nhiều chỗ, nó chỉ nêu một lần và tham chiếu tất cả vị trí bị ảnh hưởng. Điều này đặc biệt có giá trị với PR lớn, nơi nhiễu từ AI review trước đây bị khuếch đại. Cả hai tính năng đi kèm với UI changeset được cập nhật giúp apply và dismiss gợi ý nhanh hơn. Đây là một phần trong nỗ lực rộng hơn của GitHub để làm cho Copilot review cảm giác như phản hồi của senior engineer thay vì chỉ là linter.
Nếu team bạn đang dùng Copilot Code Review, hãy opt-in vào new PR experience ngay để có severity labels — nó giúp quy trình review có cấu trúc hơn và giảm đáng kể thời gian triage gợi ý AI trên PR lớn.

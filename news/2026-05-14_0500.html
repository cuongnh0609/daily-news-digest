C3Node.js 26 Enables Temporal API by Default — JavaScript's Long-Awaited Replacement for the Date Object

📰 Article (English)

Node.js 26 has shipped with the Temporal API enabled by default, marking a pivotal moment for JavaScript date and time handling. The built-in Date object has been notoriously problematic for decades: no timezone support, inconsistent parsing, millisecond-only precision, and mutable objects that cause subtle bugs. Temporal was designed from scratch to address all of these. It provides timezone-aware date objects (ZonedDateTime), immutable value semantics, nanosecond precision, clear calendar support, and a clean, chainable API surface. This aligns Node.js with Chrome 144/Edge 144 (shipped April 2026) and Firefox 139 (May 2025), which already enabled Temporal. Safari remains the only major holdout. For server-side JavaScript, this means developers can drop date manipulation libraries like date-fns, Luxon, or Moment.js in new code and use native Temporal instead. The migration path is opt-in — existing Date-based code continues to work. Node.js 26 enters a six-month active release phase before automatically transitioning to LTS status.

🇻🇳 Bản dịch + 📖 Giải thích thuật ngữ
Node.js 26 đã ra mắt với Temporal API được bật mặc định, đánh dấu một bước ngoặt quan trọng cho việc xử lý ngày giờ trong JavaScript. Object Date tích hợp sẵn lâu nay nổi tiếng đầy vấn đề: không hỗ trợ timezone, phân tích không nhất quán, chỉ có độ chính xác millisecond, và là đối tượng mutable gây ra các bug tinh vi. Temporal được thiết kế từ đầu để giải quyết tất cả những vấn đề này. Nó cung cấp đối tượng ngày giờ có timezone (ZonedDateTime), ngữ nghĩa giá trị bất biến (immutable), độ chính xác nanosecond, hỗ trợ lịch rõ ràng và API sạch có thể chain. Điều này giúp Node.js đồng bộ với Chrome 144/Edge 144 và Firefox 139 vốn đã bật Temporal. Safari vẫn là ngoại lệ lớn duy nhất. Với backend JavaScript, developer có thể bỏ các thư viện xử lý ngày như date-fns, Luxon hay Moment.js trong code mới và dùng Temporal native thay thế.
Trong các dự án Node.js mới, hãy bắt đầu dùng Temporal thay vì date-fns hoặc Luxon để giảm dependency. Với code hiện tại không cần migrate ngay — Date vẫn hoạt động bình thường.

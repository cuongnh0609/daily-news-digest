📰 Article (English)

OpenAI's Codex CLI has introduced encryption for inter-agent communications in its MultiAgentV2 system. Pull request #26210 changed the system so that when a parent agent spawns a child agent or sends it a task, the payload is encrypted before delivery — stored as InterAgentCommunication.encrypted_content while the human-readable content field is left empty. The change creates a significant auditability problem. The issue reporter argues this undermines one of the core principles of safe agentic systems: transparency of agent intent. The proposal isn't to revert encryption entirely — model-level privacy hardening is considered valuable — but to maintain a parallel, non-encrypted audit field in rollout metadata so human operators retain read access to task history. This highlights a broader tension in multi-agent system design: security and privacy goals for model inference can conflict with the observability requirements that safety and compliance teams depend on.

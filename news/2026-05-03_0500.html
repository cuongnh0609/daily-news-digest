B1自作MCPサーバーのトークン消費を9割削減する「退避パターン」――参照キーのみ渡してコンテキストを守る

📰 本文 + 和越対訳

MCPサーバーを自作する際、ファイル内容やクエリ結果をそのままコンテキストウィンドウに流し込むと、瞬く間にトークンを消費してしまう。この問題を解決するのが、記事著者が提唱する「退避パターン」だ。コアとなる考え方は、大きなデータを直接渡すのではなく、そのデータへの参照情報（URLや外部ストレージのパス）のみをAIに返し、AIがそこから自律的に取得するという設計にある。具体的にはSandbox MCPでは大容量ファイルのURL、DB Graph MCPではクエリ結果が500行を超えた場合にGoogle Spreadsheetsへ自動エクスポートしてシートURLのみを返す実装を紹介している。これらのパターンを組み合わせることでMCPスイート全体のトークン消費を70〜90%削減できたという。さらに、Google Workspace OAuthを使えばMCPのユーザー認証とWorkspaceアプリへのアクセスを一本化できる点も有用だ。

Khi tự xây dựng MCP server, việc đẩy toàn bộ nội dung file hay kết quả query vào context window sẽ tiêu tốn token rất nhanh. "Avoidance Pattern" (退避パターン) mà tác giả đề xuất giải quyết vấn đề này: thay vì truyền dữ liệu thực, chỉ trả về thông tin tham chiếu (URL hoặc đường dẫn) để AI tự lấy dữ liệu khi cần. Ví dụ cụ thể: Sandbox MCP trả về Git repo URL thay vì nội dung file; DB Graph MCP tự động export kết quả query vượt 500 dòng sang Google Spreadsheets và chỉ trả về URL. Kết hợp các pattern này, tác giả giảm được 70-90% token consumption trên toàn bộ MCP suite. Google Workspace OAuth cũng cho phép gộp authentication cho MCP user và Workspace app trong một luồng duy nhất.

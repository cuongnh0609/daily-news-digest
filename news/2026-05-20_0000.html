B3AIコーディングでsecretを漏らさないための「４層防御」——設計・pre-commit・GitHub・Claude Code Hooksの組み合わせ

📰 本文 + 和越対訳

AIコーディングツールを使った開発ではLLMがコードを自動生成するため、パスワードやAPIキーなどのsecretが意図せずリポジトリにcommitされるリスクが通常の開発より高い。筆者のながたく氏は実際にパスワードをGitHubにcommitするミスを犯した経験から、「注意力は消耗資源」との前提に立ち、機械的・決定論的な防壁による４層防御モデルを提案した。第1層は設計：secretを`.gitignore`対象ディレクトリに分離。第2層はpre-commitフック：`gitleaks`と`lefthook`を組み合わせ、日本語パターン（「パスワード: xxx」）も検出可能なカスタム設定を適用。第3層はGitHub Push Protection（パブリックリポジトリは無料）。第4層はClaude Code Hooks：`PreToolUse`フックでファイル書き込み前にgitleaksを実行する補助的な層。これらを多重に組み合わせることで、AIエージェントが生成したコードに潜むsecret漏洩を防ぐ実践的な設計を提供している。

Khi phát triển bằng công cụ AI coding, rủi ro vô tình commit password hay API key lên repository cao hơn so với lập trình thông thường do LLM tự động sinh code. Tác giả Nagataku, từ kinh nghiệm thực tế bị commit nhầm password lên GitHub, đề xuất mô hình phòng thủ 4 lớp dựa trên tiền đề "sự chú ý là tài nguyên có hạn". Lớp 1 là thiết kế: tách biệt secret vào thư mục thuộc `.gitignore`. Lớp 2 là pre-commit hook: kết hợp `gitleaks` và `lefthook` với cấu hình tùy chỉnh phát hiện cả pattern tiếng Nhật. Lớp 3 là GitHub Push Protection (miễn phí cho public repo). Lớp 4 là Claude Code Hooks: hook `PreToolUse` chạy gitleaks trước khi ghi file như lớp phòng thủ bổ sung. Kết hợp nhiều lớp này cung cấp thiết kế thực tiễn để ngăn ngừa rò rỉ secret ẩn trong code do AI agent sinh ra.

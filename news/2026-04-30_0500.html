B1Claude Code 1Mコンテキスト時代の設計判断 — 200K前提の常識をどこまで捨てるか

📰 本文 + 和越対訳

Claude CodeのMax/Team/EnterpriseプランでデフォルトになったとされるClaude Codeの1Mコンテキストウィンドウを受けて、200K時代に確立した設計判断のうち何を更新すべきかを数値付きで整理した実践記事がZennで公開された。筆者の核心的な主張は「1Mは全部入れていいではなく、入れるかどうかの判断基準が変わる」というものだ。Opus 4.6のMRCR v2ベンチマークによると、256Kトークン時に93%だった精度が1Mトークン時には76%まで低下しており、Context Rotはまだ消えていない。「Smart Zone」概念を援用すれば、1Mウィンドウでの安全圏は約400Kトークンと推定される。更新すべき設計として、Compaction戦略（400K到達まで走らせ、/compactはタスクの意味的区切りとして使う）、サブエージェント分割（コンテキスト節約目的からコスト・並列性目的へ）、CLAUDE.mdサイズ（1Mでは詳細に書ける）の3点が特に重要だ。Extended Thinkingトークンは出力単価で課金されるため、コンテキストを広げるほどコストが膨らむ点も注意が必要とされている。

Một bài viết thực hành được đăng trên Zenn đã sắp xếp các quyết định thiết kế nào cần cập nhật sau khi Claude Code mặc định hóa cửa sổ context 1M token cho các plan Max/Team/Enterprise. Luận điểm cốt lõi của tác giả là "1M không có nghĩa là cứ nhồi hết vào, mà là tiêu chí quyết định có nên nhồi hay không đã thay đổi." Theo benchmark MRCR v2 của Opus 4.6, độ chính xác giảm từ 93% ở 256K token xuống còn 76% ở 1M token — Context Rot vẫn chưa biến mất. Áp dụng khái niệm "Smart Zone", vùng an toàn thực dụng trong cửa sổ 1M được ước tính khoảng 400K token. Ba điểm thiết kế quan trọng cần cập nhật là: chiến lược Compaction (chạy đến 400K mới compact, dùng /compact như điểm phân ranh ngữ nghĩa), phân chia sub-agent (từ mục đích tiết kiệm context sang mục đích kiểm soát chi phí và song song hóa), và kích thước CLAUDE.md (với 1M có thể viết chi tiết hơn). Cũng cần lưu ý rằng token Extended Thinking bị tính theo đơn giá output, nên càng mở rộng context thì chi phí càng phình to.

B3Claude CodeのSlack MCPを複数ワークスペースに接続するハック — 公式pluginの制約を回避

📰 本文 + 和越対訳

Claude CodeでAnthropicが提供する公式Slack MCPプラグインは、OAuthクライアント単位で1ワークスペースしか接続できないという制約がある。この制限を複数ワークスペース接続で回避する方法を解説した記事がZennで公開された。問題は2層構造だ。まず、別ワークスペース用のMCPサーバーをコールバックポート3119で追加しようとすると、SlackのOAuth Appにlocalhost:3118しか登録されていないためエラーになる。次に、同じhttps://mcp.slack.com/mcpというURLで2つのサーバーを追加すると、Claude Code起動時に「同じURL」として片方がスキップされる。解決策は~/.claude.jsonを直接編集し、2つ目のサーバーURLに無意味なクエリパラメータ?ws=groupを付与することだ。Slack側では無視されて接続に影響しないが、Claude Code内での文字列比較では別URLと判定され重複排除を回避できる。自前でSlack OAuthアプリを作成する正攻法に比べ、5分程度の作業で解決できる実用的なハックだ。

Plugin Slack MCP chính thức của Anthropic cho Claude Code có hạn chế chỉ kết nối được một workspace theo từng OAuth client. Một bài viết trên Zenn đã giải thích cách vượt qua hạn chế này để kết nối nhiều workspace. Vấn đề có cấu trúc hai lớp: đầu tiên, khi thêm MCP server cho workspace thứ hai với callback port 3119, lỗi xảy ra vì Slack OAuth App chỉ đăng ký localhost:3118; tiếp theo, khi thêm hai server có cùng URL https://mcp.slack.com/mcp, Claude Code bỏ qua một cái khi khởi động vì cho là trùng. Giải pháp là chỉnh sửa trực tiếp ~/.claude.json và thêm query param vô nghĩa ?ws=group vào URL server thứ hai. Phía Slack bỏ qua param này nên kết nối không bị ảnh hưởng, nhưng Claude Code so sánh chuỗi và nhận ra đây là URL khác, từ đó tránh được bộ lọc trùng lặp. Đây là hack thực dụng giải quyết được vấn đề chỉ trong khoảng 5 phút, so với cách chính thống là tự tạo Slack OAuth App.

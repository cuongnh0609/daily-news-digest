B3git worktreeで並列Claude Code開発環境——DB・ドメインは分離、設定は共通化

株式会社ソニックガーデンのエンジニアが、git worktreeを使った並列ブランチ開発環境の構築方法を詳解した。各worktreeで分離すべきものとして、データベースの設定（worktree名を含むDB名で競合を防ぐ）とドメイン（puma-devで各worktreeに専用URLを付与）、およびポート番号の自動割当が挙げられる。一方、複数worktreeで共有すべきものとして、CLAUDE.local.mdファイルと.claude/settings.local.jsonをシンボリックリンクで一元管理する方針を採用している。Claude Codeのプロジェクトメモリはパス依存のためworktreeをまたいで機能しないという注意点も共有されており、代わりにCLAUDE.local.mdへの記述を推奨している。一連のセットアップを単一コマンド `/worktree-setup ` にまとめてClaude Codeに実行させることで、手動作業を極限まで削減している。

Một kỹ sư của SonicGarden đã chia sẻ cách xây dựng môi trường phát triển song song sử dụng git worktree. Những thứ cần tách biệt per-worktree gồm: cấu hình database (tên DB bao gồm tên worktree để tránh xung đột), domain (gán URL riêng cho mỗi worktree qua puma-dev), và gán port tự động. Trong khi đó, những thứ cần chia sẻ giữa các worktree là file CLAUDE.local.md và .claude/settings.local.json được quản lý tập trung qua symbolic link. Tác giả cũng lưu ý rằng project memory của Claude Code phụ thuộc vào path nên không hoạt động xuyên worktree, và khuyến nghị dùng CLAUDE.local.md thay thế. Toàn bộ quá trình setup được gộp vào một lệnh `/worktree-setup ` để Claude Code thực thi, giảm thiểu tối đa thao tác thủ công.

B1Spring Boot × Claude API × OpenTelemetry——LLM呼び出しのレイテンシ・コスト・トークン消費をリアルタイム可視化

📰 本文 + 和越対訳

Spring Boot 3 から Claude API を呼び出すシステムにおいて、レイテンシ・トークン消費量・コスト・失敗原因を OpenTelemetry（OTel）で計測する実装手法を解説した記事がZennで公開された。Spring Boot 3 に標準搭載されている Micrometer Observation を利用して Claude 呼び出し全体を1本の span として記録し、OTLP 経由でローカルの OTel Collector に送出する。その後 Jaeger でトレースを確認し、Prometheus と Grafana でメトリクスを可視化するフルスタックな観測パイプラインを構築できる。GenAI セマンティック規約に沿って低基数・高基数の属性を分離することで、メトリクスのカーディナリティ爆発を防ぐ設計も紹介されている。Claude API のコストや失敗パターンを本番環境で継続的に追跡したい開発者にとって、即座に応用できる実践的な内容となっている。

Một bài viết trên Zenn hướng dẫn cách tích hợp OpenTelemetry (OTel) vào ứng dụng Spring Boot 3 gọi Claude API, giúp theo dõi realtime các chỉ số quan trọng như latency, lượng token tiêu thụ, chi phí và nguyên nhân lỗi. Tác giả sử dụng Micrometer Observation — framework đo lường có sẵn trong Spring Boot 3 — để gói toàn bộ mỗi lần gọi Claude thành một span duy nhất, rồi đẩy qua OTLP tới OTel Collector cục bộ. Từ đó có thể xem trace trên Jaeger và dashboard metrics trên Prometheus + Grafana. Bài viết cũng trình bày cách phân tách thuộc tính độ cardinality thấp và cao theo chuẩn GenAI Semantic Conventions để tránh bùng nổ cardinality. Đây là tài liệu thực tiễn hữu ích cho các developer muốn giám sát chi phí và hiệu suất Claude API trong môi trường production.

Zennに投稿されたエンジニアの開発環境レポートが注目を集めている。著者はRust・バックエンド系の開発者で、Intel機からi5-13500+RTX 4060Ti 16GBへのハードウェア刷新を契機に、開発環境全体を見直した。OSはArch LinuxからNixOSへ移行。NixOSの宣言的設定により「環境の腐敗」が起きにくく、長期運用で設定の肥大化を防げると評価している。エディタはNeovimの深いカスタマイズをやめてZedに移行。「合理的に考えればCursorを使うべきだが、Zedの軽快さを選んだ」とのこと。ターミナルもAlacritty+Zedのネイティブ端末に集約し、シェルはZshからFishに変更した。最大の変化はClaude Codeを主要コーディングエージェントとして採用したこと。「既存コードベースの意図を把握させるとGPT系より優秀」とし、AIによるコーディング支援が進むほど手動のエディタ最適化のROIは下がると指摘。将来的にはApple Silicon MacとNix-Darwinへの移行、IDEに縛られないマルチエージェントワークフローエンジンの構築を目指している。

📰 Article (English)

Security firm PromptArmor disclosed a critical vulnerability in the "ChatGPT for Google Sheets" extension (185,000+ downloads in weeks) that enables a single indirect prompt injection to exfiltrate multiple workbooks, deploy phishing overlays, and inject attacker-controlled chatbots into the sidebar — all in one automated attack chain. The attack works by embedding hidden instructions (white text on white background) in an external dataset. When the victim asks ChatGPT to merge that data, the injected instructions execute silently. The malicious script accesses the victim's financial model, sends it to attacker servers, parses it for hyperlinks to other workbooks, and autonomously discovers and steals those too. A demo showed 12 workbooks exfiltrated in a single exchange. Critically, the "require human approval before edits" setting does NOT prevent this — the extraction path bypasses it entirely. Two phishing variants were also demonstrated: a sidebar overlay mimicking the ChatGPT UI to harvest credentials, and a modal popup rendering an attacker-controlled phishing site. Clicking "Stop" does not halt already-running scripts. OpenAI responded by removing the model's ability to generate Apps Script code to eliminate the core attack surface, and committed to reviewing sandboxing methodology across similar extensions. Organizations can mitigate by disabling the extension via Google Workspace Permissions.

C2Backpressure Is All You Need — A 7-Layer Automated Validation Framework for Autonomous AI Coding Agents

📰 Article (English)

Lucas F. Costa proposes a framework for making AI coding agents genuinely autonomous without sacrificing output quality. He identifies two failure modes: fully unsupervised agents (fast but low quality) and human-approval-per-step agents (safe but defeats automation benefits). His solution borrows "backpressure" from systems engineering — a concept where downstream components signal upstream they cannot accept more work — and applies it as multi-layer automated validation that forces agents to self-correct before any output reaches human reviewers. The framework defines 7 validation layers that compose into an autonomous quality loop: 1. Linting + automated tests — run continuously, not just at the end 2. Manual testing — cURL requests and Playwright browser testing run by the agent itself 3. Performance benchmarking — structured output catches regressions automatically 4. Code review agent — a dedicated reviewer agent checks readability, complexity, test coverage, and type safety 5. Planning approval — a separate agent reviews architectural plans before code is written 6. Visual/UI review — screenshot comparison against design specs 7. PR monitoring — post-PR, the system watches CI failures, conflicts, reviewer comments and addresses them automatically Costa's key insight: "Any system that relies on a human to catch the machine's mistakes will be limited by the human, not the machine." The framework ships as a public npm skill (@lucasfcosta/backpressured) with a BACKPRESSURE.md config file teams can customize.

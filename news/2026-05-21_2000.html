B-01AIエージェントに「確実性」を与えるMCPサーバーを個人開発——祝日・タイムゾーン・営業日をAPIで解決

📰 本文 + 和越対訳

AIエージェントが苦手とする「確実性が必要な計算」を解決するMCPサーバー「Thousand API」を個人開発したという記事がZennに投稿された。 祝日判定・タイムゾーン変換・営業日計算などは、LLMが推論で処理しようとすると誤りが生じやすい領域だ。 開発者はこれらの処理をAPIとして外部化し、MCPサーバー経由でClaudeやCursorから呼び出せる仕組みを構築した。 アーキテクチャは完全サーバーレス構成で、コールドスタートも最小限に抑えられている。 実際にClaudeとの連携テストでは、以前は曖昧だった「3営業日後」や「祝日を除いた翌日」といった計算が正確に処理されるようになったと報告されており、エージェントの実用性向上に貢献するアプローチとして注目を集めている。

Một bài viết trên Zenn giới thiệu "Thousand API" — một MCP server được cá nhân phát triển nhằm giải quyết vấn đề "tính toán đòi hỏi độ chính xác cao" mà AI agent thường mắc lỗi. Các tác vụ như xác định ngày lễ, chuyển đổi múi giờ, tính ngày làm việc rất dễ sai khi LLM tự suy luận. Lập trình viên đã external hóa các phép tính này thành API và xây dựng cơ chế gọi từ Claude hoặc Cursor qua MCP server. Kiến trúc là serverless hoàn toàn, giảm thiểu cold start. Trong thử nghiệm thực tế, các phép tính như "3 ngày làm việc sau" hay "ngày tiếp theo không kể lễ" vốn hay sai nay được xử lý chính xác, mở ra hướng tiếp cận giúp tăng tính thực tiễn của agent.

C-02Hugging Face: "Software Forgets — Agent Traces Are the Memory" — Store Decision Traces Alongside Code

📰 Article (English)

A Hugging Face engineering post makes the case that as AI agents write more and more of our code, we're losing crucial institutional memory: the why behind decisions. The authors argue that agent decision traces — tool calls, reasoning steps, intermediate states, and rejected alternatives — should be stored durably alongside the code they produce, not discarded after each session. Traditional git commits capture what changed but not how or why an agent arrived at that result. The proposal is to persist full agent traces to object storage (e.g., HF Buckets) linked to commits, creating a "dual history" where code history and agent reasoning history are queryable together. This lets future developers or agents understand prior decisions without re-running expensive inference, and enables post-hoc audits of agentic actions. The post draws a direct analogy to observability in distributed systems: you wouldn't run a production service without traces — why would you run a production codebase without agent traces? The authors call this a prerequisite for safely scaling agentic development teams.

B-1Agent Runtimeを実装してState Machineと再会した——LLMの「状態溶解」問題を外部化で解決

📰 本文 + 和越対訳

Zennのエンジニア mofuteq 氏が公開した技術記事によると、AIエージェントシステムを構築する際に最も困難な問題の一つが「状態溶解」だという。LLM内部で処理が行われると、検索が完了したか、安全チェックを通過したか、どの段階で失敗したかといった状態情報が外部から参照できなくなる。UIで「Thinking...」とだけ表示するのはこの設計上の失敗の現れだと同氏は指摘する。解決策として、従来のState Machine（状態機械）パターンをAIエージェントに適用し、各ステップを明示的なノードとして定義することで、エージェントの動作をオペレーショナルに可視化する手法を提案している。LOAD → FILTER → SCOPE_CHECK → GENERATE → REFLECT → SAFETY_CHECK → OUTPUTという設計により、UIへの状態表示、Langfuseなどのトレースシステムとの連携、デバッグの容易化が実現できるとしている。

Kỹ sư mofuteq trên Zenn đã chia sẻ bài viết kỹ thuật về vấn đề "state dissolution" — trạng thái bị hòa tan bên trong LLM. Khi agent xử lý, các thông tin trạng thái như "đã retrieval chưa?", "đã qua safety check chưa?", "thất bại ở bước nào?" đều không thể quan sát từ bên ngoài. Hiện tượng UI chỉ hiện "Thinking..." là biểu hiện của lỗi thiết kế này. Giải pháp được đề xuất là áp dụng mô hình State Machine truyền thống vào AI agent, định nghĩa mỗi bước xử lý thành các node rõ ràng như LOAD → FILTER → SCOPE_CHECK → GENERATE → REFLECT → SAFETY_CHECK → OUTPUT, giúp quan sát trạng thái agent theo thời gian thực, tích hợp với các hệ thống trace như Langfuse và dễ debug hơn.

Superset, a Y Combinator S26 company, launched an open-source Electron desktop application that serves as an orchestration layer for running multiple CLI-based AI coding agents in parallel. The tool addresses a core bottleneck in AI-assisted development: current agents like Claude Code and OpenAI Codex CLI can only work on one task at a time, and running multiple instances manually leads to git conflicts and context confusion. Superset solves this by assigning each agent its own isolated git worktree — a separate branch and working directory — so 10 or more agents can work simultaneously without interfering. The app includes a built-in diff viewer, agent status monitoring with desktop notifications, and workspace presets for environment setup. Supported agents include Claude Code, OpenAI Codex CLI, Amp Code, Cursor Agent, Gemini CLI, GitHub Copilot, and any terminal-based CLI agent. The stack is Electron + React + Bun + Turborepo. The project received 89 HN points on launch day and is licensed under Elastic License 2.0. This represents a new category of tooling: not a coding agent itself, but an agent orchestration desktop environment.

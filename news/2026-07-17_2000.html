B-01SQL MCP Server が GA——AIエージェントが安全にデータベースを操作できる新時代へ

📰 本文 + 和越対訳

Microsoftが開発するData API Builderの一部として提供されてきたSQL MCP Serverが、2026年6月に正式版（GA）としてリリースされた。このツールはModel Context Protocol（MCP）に準拠しており、AIエージェントがデータベースを直接操作できる標準インターフェースを提供する。デフォルトでは、レコードの作成・読取・更新・削除、ストアドプロシージャの実行、集計クエリの7種類のDMLツールが公開される仕組みだ。セキュリティ面では、設定ファイル（dab-config.json）で各ツールの公開を制御でき、参照専用に制限することも可能。複雑なビジネスロジックはストアドプロシージャとして実装してカスタムツールとして登録する運用が推奨されており、AIエージェントが不正なクエリを実行するリスクを抑えながら柔軟な活用が期待される。Azure Container Appsへのデプロイにも対応する。

SQL MCP Server — một thành phần của Data API Builder do Microsoft phát triển — đã chính thức phát hành phiên bản GA vào tháng 6 năm 2026. Công cụ này tuân thủ giao thức Model Context Protocol (MCP), cung cấp giao diện chuẩn để các AI agent có thể trực tiếp thao tác cơ sở dữ liệu. Mặc định, 7 loại công cụ DML được mở: tạo, đọc, cập nhật, xóa bản ghi, thực thi stored procedure, và truy vấn tổng hợp. Về bảo mật, file cấu hình (dab-config.json) cho phép kiểm soát từng công cụ được mở ra, kể cả hạn chế chỉ cho phép đọc. Thực hành được khuyến nghị là triển khai logic nghiệp vụ phức tạp dưới dạng stored procedure và đăng ký như custom tool, giúp giảm thiểu rủi ro AI agent thực thi truy vấn trái phép. Công cụ cũng hỗ trợ deploy lên Azure Container Apps.

米カリフォルニア工科大学発のAIスタートアップ・PrismMLが、「スマートフォン上で稼働する初の27Bパラメーターモデル」と銘打った大規模言語モデル「Bonsai 27B」を発表した。通常、27Bモデルは16ビット精度で約54GBものメモリを必要とするが、Bonsai 27Bは各パラメーターの重みを1〜2ビット程度まで削減する量子化技術によって大幅なサイズ縮小を実現した。1-bit版はわずか3.9GB（iPhone対応）、Ternary版は5.9GB（ノートPC向け）で、それぞれ「−1」「＋1」と「−1」「0」「＋1」で重みを表現する。Qwen3.6 27Bをベースとしたマルチモーダル対応モデルであり、15の標準ベンチマークでTernary版はフル精度の95%、1-bit版は90%の性能を維持している。Apache 2.0ライセンスでオープンソース公開されており、エッジAIの実用化に向けた注目の一歩となっている。

Startup AI PrismML, xuất phát từ Caltech (Mỹ), đã công bố mô hình ngôn ngữ lớn "Bonsai 27B" — được giới thiệu là mô hình 27 tỷ tham số đầu tiên có thể chạy trên smartphone. Thông thường, mô hình 27B cần khoảng 54GB bộ nhớ ở độ chính xác 16-bit, nhưng Bonsai 27B đạt được kích thước nhỏ gọn đáng kể nhờ kỹ thuật lượng tử hóa (quantization) giảm trọng số mỗi tham số xuống chỉ còn 1-2 bit. Phiên bản 1-bit chỉ nặng 3,9GB (tương thích iPhone), phiên bản Ternary nặng 5,9GB (dành cho laptop), biểu diễn trọng số bằng "−1", "+1" và "−1", "0", "+1" tương ứng. Mô hình đa phương thức dựa trên Qwen3.6 27B này đạt 95% (Ternary) và 90% (1-bit) hiệu suất so với full precision trên 15 benchmark tiêu chuẩn. Được phát hành mã nguồn mở theo giấy phép Apache 2.0, đây là bước tiến đáng chú ý hướng tới thực tế hóa AI ở thiết bị biên.

C-01The Human-in-the-Loop Is Tired: Why AI Coding Tools Are Eroding Developer Satisfaction

📰 Article (English)

Laura Summers writing for Pydantic argues that LLM-assisted programming has created a paradoxical situation: the work is measurably more productive yet fundamentally less satisfying. She identifies this as "the human reward function problem" — AI has automated the gratifying parts of coding (solving puzzles, understanding logic, feeling clever) and replaced them with exhausting supervision and review work. The satisfying loop of problem → effort → solution → reward has been broken; what remains is the cognitive load of overseeing an output that feels partially yours. Summers draws a parallel to how responsive design disrupted web design in the 2010s — it felt threatening at first, but actually elevated the craft by removing mechanical work. She argues the engineering profession is similarly evolving: the bottleneck has shifted from code-writing to human judgment, architectural vision, and taste — skills that are genuinely scarce. Developers feel overwhelmed not because they are becoming obsolete, but because their professional identity hasn't caught up with the new role. The piece closes with a call to reframe what "being good at your job" means in an AI-augmented context: not writing more lines faster, but knowing which lines matter, when to delegate, and what systems to build in the first place.

C-02GitHub Code Scanning Now Flags AI Prompt Injection Attacks in Pull Requests

📰 Article (English)

GitHub has extended its Code Scanning feature to detect AI prompt injection vulnerabilities directly within pull requests, shipping alongside CodeQL 2.26.0. The new capability introduces a dedicated query — js/system-prompt-injection — that statically analyzes JavaScript and TypeScript code for patterns where user-controlled input can reach an LLM's system prompt without proper sanitization. The detection targets a class of vulnerability that has become increasingly critical as more applications embed AI agents: if an attacker can inject instructions into a system prompt, they can redirect the agent's behavior, exfiltrate data, or bypass security controls. The scanning surfaces findings in the standard Security tab and as inline annotations on affected pull request diff lines, integrating into existing CI/CD review workflows with no additional configuration required for repos already running CodeQL. The query covers common patterns including template literal interpolation, string concatenation, and framework-specific prompt construction APIs. GitHub notes that the feature is part of a broader push to bring "AI security" into the standard developer security toolkit, alongside existing detections for SQL injection, XSS, and command injection. Organizations on GitHub Advanced Security get the feature automatically; free public repos benefit from default CodeQL setup.

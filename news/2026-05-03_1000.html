B1TypeScriptワークフローの「正しさ」をLean定理証明で検証――AI Shiftが示すハイブリッド手法

📰 本文 + 和越対訳

AI Shiftのエンジニアが、TypeScriptで実装したワークフローエンジンの正しさを定理証明支援系「Lean」とランダムテストを組み合わせて検証するハイブリッドアプローチを公開した。手法の核心は3段階構成だ。まずLeanでワークフローの仕様を数学的モデル（Node → Option Node型の関数）として形式化し、次に「各ノードの出力先は高々1つ」という性質をLean上で形式的に証明する。最後に、TypeScript実装とLeanモデルの両方にランダム入力を与え、出力の差分を継続的に検出するテストをCIに組み込む。重要な点として、Leanが保証するのはあくまで形式仕様の性質であり、実装そのものの保証はランダムテストが担う。この分離設計により「証明の複雑さ」と「実装の実用性」を両立させており、フォーマル検証を本番コードに現実的に適用する事例として注目を集めている。

Một kỹ sư tại AI Shift đã công bố phương pháp hybrid kết hợp hệ thống chứng minh định lý "Lean" và random test để xác minh tính đúng đắn của workflow engine viết bằng TypeScript. Phương pháp gồm 3 bước: đầu tiên hình thức hóa đặc tả workflow trong Lean dưới dạng mô hình toán học (hàm kiểu `Node → Option Node`), tiếp theo chứng minh hình thức tính chất "mỗi node có tối đa một output" trên Lean, và cuối cùng tích hợp vào CI một test liên tục phát hiện sai lệch bằng cách cho cùng input ngẫu nhiên vào cả implementation TypeScript lẫn model Lean. Điểm quan trọng là Lean chỉ đảm bảo tính chất của đặc tả hình thức, còn việc đảm bảo implementation thực tế là trách nhiệm của random test. Thiết kế tách biệt này vừa đạt được "độ phức tạp của chứng minh" lại vừa đảm bảo "tính thực dụng của implementation", thu hút sự chú ý như một ví dụ thực tế áp dụng formal verification vào production code.

C2The Agent Harness Belongs Outside the Sandbox — A Production Architecture for Multi-User AI Agents

📰 Article (English)

Andrea Luzzardi at Mendral published a detailed architectural argument for why the agent harness — the orchestration loop that drives an LLM's tool-call cycle — should run outside the sandbox rather than inside it. The canonical "inside sandbox" pattern (e.g., Claude Code SDK, local agent frameworks) bundles the loop, LLM calls, and tool execution inside a single container. Mendral's production system instead keeps the loop on backend infrastructure and invokes the sandbox on-demand via API. This enables three critical capabilities: (1) credentials and secrets never enter the sandbox, reducing blast radius; (2) sandboxes can be suspended during LLM "thinking" time and resume in 25 ms via Blaxel, cutting compute costs dramatically; (3) durable execution via Inngest checkpoints each turn, so agent sessions survive server restarts or deploys. The most innovative piece is a virtualized filesystem: file I/O is intercepted at the path level — workspace operations go to the sandbox, while memory and skill file reads are transparently routed to PostgreSQL. This preserves the familiar file interface that models were trained on while sharing org-level knowledge across parallel sessions. The author acknowledges current limitations: many new agent capabilities assume a local filesystem, bash commands bypass the interception layer, and last-writer-wins concurrency is insufficient for truly collaborative agents.

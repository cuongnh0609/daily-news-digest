📰 本文 + 和越対訳

2025年3月、Anthropicが誤ってClaude CodeのTypeScriptソースコード（1,906ファイル・51.2万行）をnpmに公開してしまったインシデントがあった。著者はそのコードを分析し、プロダクション品質のAIエージェントを構成する5つの設計原則を見出した。第一に、約52個の専用ツールには「USE WHEN」「DO NOT USE」条件が組み込まれており、ガバナンスをツール定義そのものに埋め込んでいる。第二に、クエリの複雑度に応じてエージェントループを迂回する分岐ロジックを採用している。第三に、プロンプトをキャッシュ可能な静的セクションと動的セクションに分離してコストを72%削減している。第四に、MEMORY.mdという約500トークンのインデックスで詳細ファイルへのポインタを管理する階層的メモリ設計を採用している。第五に、セキュリティ制約をツール定義内に組み込み、判断と実行を分離している。結論として「品質の差はアルゴリズムではなくハーネスの品質から生まれる」という示唆が得られる。

Năm 2025, Anthropic vô tình công bố mã nguồn TypeScript đầy đủ của Claude Code (1.906 file, hơn 512.000 dòng) lên npm. Thứ nhất, khoảng 52 tool chuyên dụng tích hợp điều kiện "USE WHEN" và "DO NOT USE" ngay trong phần mô tả. Thứ hai, logic phân nhánh theo độ phức tạp của query giúp bỏ qua agent loop cho câu hỏi đơn giản. Thứ ba, tách prompt thành phần static có thể cache và phần dynamic, giảm chi phí 72%. Thứ tư, thiết kế bộ nhớ phân cấp với MEMORY.md (~500 token) chỉ lưu con trỏ tới các file chi tiết. Thứ năm, nhúng ràng buộc bảo mật vào định nghĩa tool, tách biệt phán quyết với thực thi. Bài học cốt lõi: "chất lượng AI agent đến từ chất lượng harness, không phải thuật toán."

B3むしろ --dangerously-skip-permissions の方が安全かも？ — Hooksで確認疲れを解消

📰 本文 + 和越対訳

著者のチームはClaude Codeを複数インスタンスで並列実行することが多く、次々と押し寄せる実行確認に「無意識にYesを押してしまう確認疲れ」という問題を抱えていた。Dockerサンドボックスは認証情報やファイル同期の手間が大きく実用的でなく、.claude/settings.jsonの権限設定も信頼性に疑問があった。解決策は「PreToolUse Hookで危険コマンドを厳密にブロックした上で、--dangerously-skip-permissionsフラグを有効化する」という逆説的アプローチだ。公式ドキュメントによれば、HookのpermissionDecision: 'deny'はbypassPermissionsモードでも確実に動作する。チームはenforce-permissions.shスクリプトを作成し、rm・sudo・cdk destroyなど30種以上の危険パターンをブロック、git push・terraform applyなど15種以上を要確認とした。88件のテストケースで実装を検証し、朝に起動してタスクを委ねた結果を夕方に確認する非同期ワークフローを実現した。

Nhóm tác giả thường chạy nhiều instance Claude Code song song và gặp vấn đề "confirmation fatigue" — vô thức bấm Yes mà không xem xét kỹ. Docker sandbox bất tiện do quản lý credential và đồng bộ file, còn cài đặt permission trong `.claude/settings.json` cũng thiếu độ tin cậy. Giải pháp là cách tiếp cận nghịch lý: dùng PreToolUse Hook để chặn lệnh nguy hiểm một cách nghiêm ngặt, rồi bật flag `--dangerously-skip-permissions`. Theo tài liệu chính thức, `permissionDecision: 'deny'` trong Hook hoạt động ngay cả trong chế độ `bypassPermissions`. Nhóm tạo script `enforce-permissions.sh` chặn 30+ pattern nguy hiểm (rm, sudo, cdk destroy...) và yêu cầu xác nhận 15+ thao tác nhạy cảm (git push, terraform apply...). Sau khi kiểm thử 88 test case, họ đạt được workflow bất đồng bộ: buổi sáng khởi động agent giao việc, tối xem kết quả.

The project is noteworthy because it demonstrates that careful harness engineering — not the underlying model — is the primary driver of agent performance.

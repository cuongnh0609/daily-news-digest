C-3Simon Willison: Running Python Code in a Sandboxed WASM Environment Using MicroPython

📰 Article (English)

Simon Willison has released micropython-wasm (v0.1a2), a novel approach to sandboxed Python execution aimed at safely running untrusted code in AI agent and plugin contexts. The core problem: plugin systems in Datasette, his LLM CLI tool, and sqlite-utils all run with full host privileges, creating security risks when executing buggy or malicious code submitted by users or AI agents. WebAssembly emerged as the right isolation primitive — battle-tested in browsers for a decade, with built-in memory isolation, CPU limits, and no filesystem access by default. Rather than the full Pyodide interpreter (browser-only, heavyweight), Willison chose lightweight MicroPython compiled to WASM, integrated via the wasmtime Python library. The implementation runs sandboxed code in a persistent thread communicating via request/reply queues, which allows stateful execution across multiple calls without the overhead of restarting the WASM runtime each time. Host functions are exposed through a 362KB WASM blob. The sandbox is intentionally restrictive: no file access, bounded memory and CPU. Willison notes this is still alpha — he doesn't recommend it for high-stakes deployments yet — but he's actively stress-testing it against AI agents attempting escape, positioning it as infrastructure for safe agentic tool execution in long-running pipelines.

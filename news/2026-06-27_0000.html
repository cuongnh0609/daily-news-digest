B1Claude Code × OpenTelemetry——Splunkでエージェント・スキル・モデル別コストをリアルタイム可視化する方法

📰 本文 + 和越対訳

Claude Codeでサブエージェントやスキルを複数並走させているとき、「どのエージェントがいくらトークンを消費しているか」を把握するのは難しい。この記事では、OpenTelemetryを使ってClaude Codeの計装を行い、Splunk Observability Cloudにメトリクス・トレース・ログを送信することで、エージェント別・スキル別・モデル別のコストをダッシュボードで確認できる環境を構築する方法を詳解している。設定の核心はClaude Codeのsettings.jsonにCLAUDE_CODE_ENABLE_TELEMETRY=1とCLAUDE_CODE_ENHANCED_TELEMETRY_BETA=1を追加するだけで、後はDockerで動かすSplunk OpenTelemetry Collectorがメトリクスを受け取り転送する。さらに、Splunk REST APIを使ったダッシュボード・アラートの自動生成コードも公開されており、コスト暴走の即時検知まで実現している。

Khi chạy nhiều subagent và skill song song trong Claude Code, việc theo dõi mức tiêu thụ token của từng agent là rất khó khăn. Cấu hình cốt lõi chỉ cần thêm `CLAUDE_CODE_ENABLE_TELEMETRY=1` và `CLAUDE_CODE_ENHANCED_TELEMETRY_BETA=1` vào `settings.json` của Claude Code, sau đó Splunk OpenTelemetry Collector chạy bằng Docker nhận và chuyển tiếp metrics. Ngoài ra, bài viết còn chia sẻ code tự động tạo dashboard và alert qua Splunk REST API để phát hiện ngay khi chi phí vượt ngưỡng.

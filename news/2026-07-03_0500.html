The methodology shows how to close the prompt-engineering loop in a rigorous way: build a testing harness where DSPy agents invoke the agent's actual tools against a live in-process Datasette instance, generate a gold-standard evaluation dataset automatically, define custom metrics, then let DSPy search for better prompts. The proposed fix: either include column names directly in the schema listing, or remove the conflicting instruction. The larger lesson is that DSPy's optimizer can find failure modes in prompt logic that manual inspection misses — and that building a proper eval harness for your agent before optimizing prompts is not optional, it's the only way to know if changes help.

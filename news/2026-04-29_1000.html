B-01Claude CodeのPermission確認を97%自動化するOSS「ccgate」— LayerXが公開

📰 本文 + 和越対訳

LayerXのエンジニアTAK848氏が、Claude Codeなどのコーディングエージェントにおけるツール実行前のPermission確認を自動化するCLIツール「ccgate」をOSSとして公開した。コーディングエージェントは実行ごとに「このToolを実行していいですか？」という確認を求めるが、繰り返し同様の判断を求められることで開発者の集中力が削がれるという課題がある。ccgateはエージェントが確認を出そうとした瞬間に割り込み、別のLLMに可否判定を代行させる仕組みだ。判定結果は「許可」「理由付き拒否」「ユーザーへの差し戻し」の3パターンで、設定はjsonnet形式の自然言語ルールで記述できる。著者の実環境では2026年4月1〜28日のおよそ2,000件の確認のうち、97%が自動化されたと報告されており、Claude CodeのAuto Modeを補完するツールとして注目されている。拒否時に理由を返すことでエージェントが適切に軌道修正できる点も評価されている。

Kỹ sư TAK848 của LayerX đã công bố OSS "ccgate" — một công cụ CLI tự động hóa việc xác nhận Permission trước khi thực thi tool trong các coding agent như Claude Code. Vấn đề hiện tại là agent yêu cầu xác nhận "có được thực thi tool này không?" liên tục, làm phân tâm developer. ccgate can thiệp vào đúng thời điểm agent sắp hỏi xác nhận, rồi giao cho một LLM khác quyết định thay. Kết quả phán quyết có 3 loại: "cho phép", "từ chối kèm lý do", và "trả lại cho người dùng". Cấu hình viết bằng quy tắc ngôn ngữ tự nhiên ở định dạng jsonnet. RCE (Remote Code Execution) /ɑːr siː iː/ Thực thi mã từ xa — lỗ hổng cho phép kẻ tấn công chạy code tùy ý trên server mục tiêu mà không cần quyền truy cập trực tiếp. Được coi là loại lỗ hổng nghiêm trọng nhất
push option injection /pʊʃ ˈɒpʃən ɪnˈdʒɛkʃən/ Tấn công injection qua push option của git — dữ liệu người dùng không được lọc và được embed trực tiếp vào cấu trúc nội bộ của server, cho phép chèn lệnh độc hại
path traversal /pɑːθ ˈtrævərsl/ Leo thang đường dẫn — kỹ thuật tấn công dùng "../" hoặc ký tự tương tự để truy cập thư mục ngoài phạm vi được phép, thường để đọc hoặc ghi file nhạy cảm
git hook /ɡɪt hʊk/ Script tự động chạy tại các sự kiện của git (pre-commit, post-receive...). Server-side hook post-receive chạy sau khi nhận push — nếu bị inject, có thể thực thi lệnh tùy ý
sanitization /ˌsænɪtaɪˈzeɪʃən/ Lọc/làm sạch dữ liệu đầu vào — loại bỏ hoặc escape các ký tự đặc biệt để ngăn injection. Thiếu sanitization là nguyên nhân của phần lớn lỗ hổng web security

B1Claude Code を自律エージェント基盤として運用する——context rot から逆算する設計

📰 本文 + 和越対訳

Claude Code を単なるコード補完ツールとしてではなく、自律エージェントのインフラとして運用する設計論が注目されている。著者が提唱するのは「context rot（コンテキスト劣化）から逆算する設計」という考え方だ。コンテキストウィンドウが埋まるにつれてモデルの挙動が変化し、CLAUDE.md の指示への遵守率が低下していく現象を「context rot」と呼び、これに対処するための構造的なアプローチを体系化している。具体的には、CLAUDE.md を200行以内に抑えること、Hooks をCIライクな強制機構として使うこと、そして並列化の方針としてサブエージェント・エージェントチーム・ダイナミックワークフローの3種を使い分けることが推奨されている。また6月15日の課金体系変更に伴い、プログラム利用（claude -p、SDK、GitHub Actions）がインタラクティブなサブスクリプションとは別のクレジットプールに移行することも解説されており、コスト設計の面でも実践的な知見が詰まった内容となっている。

Một bài phân tích đang thu hút sự chú ý về việc vận hành Claude Code không phải như một công cụ hoàn thiện code thông thường mà như hạ tầng cho agent tự chủ. Tác giả đề xuất phương pháp thiết kế "đảo ngược từ context rot" (sự suy giảm hiệu suất khi context window đầy). Khi context window lấp đầy, tỷ lệ tuân thủ hướng dẫn trong CLAUDE.md giảm dần - hiện tượng được gọi là "context rot". Để xử lý vấn đề này, bài viết đề xuất: giữ CLAUDE.md dưới 200 dòng, dùng Hooks như cơ chế kiểm soát cưỡng bức kiểu CI, và phân biệt ba chiến lược song song hóa: subagents, agent teams, dynamic workflows. AIエージェントが実世界のシステムとして成熟していく過程を、Chat Agent → Tool Agent → Runtime Agent → Managed Agent という4段階で整理した論考が公開された。単なる成熟度モデルではなく、現実のリスク圧力によって各段階への移行が促されるという「リスク圧力モデル」として説明されている。Chat Agent はメッセージと model call の基本ループを管理するだけだが、Tool Agent はツールスキーマや権限チェックを伴う実行パイプラインへと進化する。Runtime Agent はターン制限・タイムアウト・エラー回復・セッションログ等を必要とし、Managed Agent では複数ユーザー・複数プロジェクトを扱うサンドボックス分離やガバナンスポリシーが求められる。Harness（制御システム全体）は最初から設計するものではなく、実際の制約に応じて有機的に成長するものだという洞察は、現場エンジニアにとって実践的な指針となっている。

Một bài phân tích về con đường tiến hóa của AI agent trong thực tế đã được công bố, phân loại theo 4 giai đoạn: Chat Agent → Tool Agent → Runtime Agent → Managed Agent. Đây không chỉ là mô hình đánh giá mức độ trưởng thành, mà là "mô hình áp lực rủi ro" - mỗi giai đoạn được kích hoạt bởi các ràng buộc thực tế. Chat Agent chỉ quản lý vòng lặp messages và model calls cơ bản. Tool Agent tiến hóa thêm tool schemas và pipeline thực thi kèm kiểm tra quyền hạn. Runtime Agent cần giới hạn lượt, timeout, phục hồi lỗi và session logs. Managed Agent đòi hỏi sandbox isolation và governance policies cho đa người dùng, đa dự án. Điểm sâu sắc nhất là Harness (hệ thống kiểm soát toàn bộ) không được thiết kế từ đầu mà phát triển hữu cơ theo các ràng buộc thực tế.

C1Microsoft Open-Sources pg_durable — Durable Execution Engine Built Directly into PostgreSQL

📰 Article (English)

Microsoft has open-sourced pg_durable, a PostgreSQL extension that brings durable execution directly into the database. Rather than orchestrating long-running tasks across external services like separate job queues or cron systems, developers can now define workflows in SQL and let PostgreSQL handle checkpointing and recovery natively. The extension treats workflows as directed graphs of SQL steps, persisting execution state to the database after each step. If a crash or restart occurs, execution resumes from the last checkpoint rather than restarting from scratch. Under the hood, pg_durable leverages two Rust libraries—duroxide (orchestration runtime) and duroxide-pg (PostgreSQL state provider)—running inside the database server via the pgrx framework. It supports PostgreSQL 17 and 18. The project targets use cases like data pipelines, scheduled maintenance, vector embedding workflows, and fan-out aggregation—all patterns where data already lives in Postgres. The composable ~> and |=> SQL operators allow declarative workflow definition without any external orchestration infrastructure. The project is currently in preview, available as Debian packages or source builds.

C3Simon Willison Releases datasette-agent-micropython — Python Agent Tools Running Inside SQLite via WASM Sandbox

📰 Article (English)

Simon Willison released datasette-agent-micropython 0.1a0, a plugin that enables AI agents to execute arbitrary Python code tools directly within Datasette — a Python-based data exploration tool — using MicroPython compiled to WebAssembly as a secure sandbox. This is part of Willison's broader micropython-wasm project, which reached version 0.1a2 this week. The technical architecture is notable: rather than running Python code in the host process (which would be unsafe), the plugin routes tool execution through MicroPython running inside a WASM sandbox. This provides strong isolation — agent-written code cannot escape to the host system, access arbitrary files, or make unauthorized network calls. The approach directly addresses a critical challenge in production agent deployment: how to allow agents to write and execute dynamic code tools without introducing unacceptable security risks. Willison's design essentially gives agents a safe "scratch pad" for Python computation within the context of Datasette's SQL data interface, opening the door to agents that can write ad-hoc data analysis scripts, manipulate query results, and generate custom visualizations — all sandboxed from the host environment.

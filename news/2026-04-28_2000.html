B-01LLMフレンドリーなコードベース設計 — AIエージェントが「迷子」にならない構造の作り方

📰 本文 + 和越対訳

Claude CodeなどのAIエージェントを使った開発において、エージェントが大量の無関係なファイルを開いてしまい、精度が落ちる「迷子問題」が顕在化している。この根本原因はモデル性能ではなくコードベースの構造にあると著者は主張する。提唱されるのは3つの設計原則：①境界の明確化（モジュール間の依存を最小化）、②命名の自己説明性（ファイル・関数名で役割が一目でわかる）、③変更の局所性（1つの修正が波及するファイル数を最小化）。arXivの研究ではCLAUDE.mdやAGENTS.mdの導入だけでエージェントのランタイムが28.6%短縮、トークン消費が16.6%削減されたという。「LLMのために設計を歪める必要はない」という結論も興味深く、人間にとって読みやすいコードはLLMにとっても読みやすいことが実証されている。

Khi sử dụng các AI agent như Claude Code để phát triển phần mềm, vấn đề "agent bị lạc" ngày càng nổi cộm — agent mở quá nhiều file không liên quan và giảm độ chính xác. Tác giả lập luận rằng nguyên nhân gốc rễ nằm ở cấu trúc codebase, không phải hiệu năng mô hình. Ba nguyên tắc thiết kế được đề xuất: ①phân định ranh giới rõ ràng giữa các module, ②đặt tên tự mô tả (tên file/hàm nói lên vai trò), ③giảm thiểu phạm vi lan toả khi thay đổi. Nghiên cứu trên arXiv cho thấy chỉ việc thêm file CLAUDE.md hay AGENTS.md đã giúp rút ngắn runtime agent 28,6% và giảm tiêu thụ token 16,6%. Kết luận quan trọng: code dễ đọc với con người cũng dễ đọc với LLM — không cần bóp méo thiết kế vì AI.

Dù AI như Claude Code hay Cursor đẩy nhanh tốc độ phát triển, nhưng "lý do vì sao" lại ngày càng khó nhìn thấy. Ví dụ thực tế: dùng Vercel Analytics Drains lưu dữ liệu hành vi người dùng vào DB rồi đối chiếu với output của AI.

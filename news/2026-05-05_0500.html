B1MCPの攻撃面を理解する——ツール汚染・rug pull・シャドーイング攻撃の実態

📰 本文 + 和越対訳

MCP（Model Context Protocol）を本番環境に導入する前に確認すべきセキュリティリスクが、Zennコミュニティで注目を集めている。主要な脅威として「ツール汚染（Tool Poisoning）」「rug pull（サーバー側の事後的な動作変更）」「シャドーイング（既存ツール名を偽装した悪意あるMCPサーバーの注入）」の3パターンが挙げられている。特にツール汚染はAIエージェントの推論過程を直接操作できるため危険度が高い。過去60日間でMCP関連のCVE（共通脆弱性識別子）が30件以上報告されており、OWASP MCP Top 10に相当する独自のチェックリストを公開する動きも出てきた。本番MCPサーバーを運用するには、ツール実行のサンドボックス化・権限の最小化・ログ監査の徹底が不可欠とされる。

Các rủi ro bảo mật cần kiểm tra trước khi triển khai MCP (Model Context Protocol) vào môi trường production đang được cộng đồng Zenn chú ý. Ba mối đe dọa chính được nêu ra là: "Tool Poisoning" (nhiễm độc công cụ), "rug pull" (thay đổi hành vi server sau khi đã triển khai), và "shadowing" (injection MCP server độc hại giả mạo tên tool có sẵn). Đặc biệt, Tool Poisoning nguy hiểm vì có thể thao túng trực tiếp quá trình suy luận của AI agent. Trong 60 ngày qua đã có hơn 30 CVE liên quan MCP được báo cáo, và một số cộng đồng bắt đầu công bố checklist riêng tương đương OWASP MCP Top 10. Để vận hành MCP server production an toàn, cần sandbox hóa tool execution, tối thiểu hóa quyền hạn, và giám sát log đầy đủ.

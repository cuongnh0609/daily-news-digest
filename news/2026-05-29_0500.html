B1Claude Opus 4.8発表——Dynamic Workflowsで数百の並列サブエージェント、Effort Controlで速度と品質を制御

📰 本文 + 和越対訳

Anthropicは28日、フラッグシップモデル「Claude Opus 4.8」を発表した。コーディング、推論、知識作業全般にわたって前世代のOpus 4.7を上回る性能を示し、コードの欠陥を見落とす確率は約4分の1に低下したとされる。注目の新機能として「Dynamic Workflows」が追加され、数十万行規模のコードベース移行など大規模タスクで数百の並列サブエージェントを実行できる。また「Effort Control」機能では、ユーザーが処理の深さと速度のバランスを調整でき、レートリミットの消費量を最適化できる。Messages APIでは会話途中でシステムプロンプトをキャッシュを壊さずに更新できる機能も追加。さらに「Claude Mythos」と呼ばれる上位インテリジェンスクラスのモデルが限定公開されており、近日中の一般公開が予告されている。

Anthropic ngày 28 công bố mô hình hàng đầu Claude Opus 4.8, vượt trội so với Opus 4.7 trong các lĩnh vực lập trình, suy luận và xử lý tri thức. Xác suất bỏ sót lỗi trong code giảm xuống còn khoảng 1/4 so với trước. Tính năng nổi bật nhất là "Dynamic Workflows" cho phép thực thi hàng trăm subagent song song cho các tác vụ quy mô lớn như di chuyển codebase hàng chục nghìn dòng. "Effort Control" cho phép người dùng điều chỉnh độ sâu xử lý và tốc độ phản hồi, giúp tối ưu lượng rate limit tiêu thụ. Messages API cũng bổ sung khả năng cập nhật system prompt giữa chừng mà không làm mất cache. Ngoài ra, mô hình cao cấp hơn "Claude Mythos" đang được thử nghiệm giới hạn và sắp được công bố rộng rãi.

GitHub and npm shipped two supply-chain security improvements in npm CLI v11.15.0+. Staged Publishing is now generally available: instead of a package becoming instantly installable upon npm publish, maintainers run npm stage publish, which places the tarball into a queue.

📰 本文 + 和越対訳

エンジニアのLily氏が公開した記事では、Claude Codeの開発環境を人間の介入なしに毎日自律改善するautopilotシステムの設計が詳述されている。このシステムは毎朝5時に起動し、claude -pコマンドによるheadlessモードで~/.claude/配下のファイルのみを対象に1タスクずつ改善を行う。重要なのは、暴走を防ぐための二重のガード機構だ。コスト管理では、残トークン量を数値で検証し（ステータスラベルだけでは不十分）、壁時計による2時間タイムアウトを設けている。また、タスクの重複実行を防ぐためJSONL形式の履歴管理を採用し、48時間以内に完了したタスクや2回連続で失敗したタスクはスキップする。さらに、Claudeが自己報告するメトリクスをディスク変更量や改変ファイル数と照合する検証レイヤーを実装することで、主張と実際のシステム変化の乖離を検知できるようにしている。この設計は、自律的なLLMシステムに数値的な閾値・計時・履歴追跡・独立検証が必要であることを実証している。

Kỹ sư Lily đã công bố bài viết mô tả chi tiết thiết kế hệ thống autopilot tự động cải thiện môi trường Claude Code mà không cần sự can thiệp của con người. Hệ thống khởi động lúc 5 giờ sáng, chạy ở chế độ headless qua lệnh claude -p, mỗi lần chỉ thực hiện một task cải thiện trong thư mục ~/.claude/. Điểm quan trọng là cơ chế bảo vệ kép để ngăn agent "bạo loạn": quản lý chi phí bằng cách kiểm tra số lượng token còn lại theo giá trị số thực (không chỉ dựa vào nhãn trạng thái), đặt timeout 2 tiếng theo đồng hồ thực. Để chống thực hiện task trùng lặp, hệ thống dùng lịch sử định dạng JSONL và bỏ qua các task đã hoàn thành trong 48h hoặc thất bại 2 lần liên tiếp. Ngoài ra, một lớp kiểm tra độc lập so sánh metric Claude tự báo cáo với thay đổi thực tế trên disk, giúp phát hiện sự sai lệch giữa tuyên bố và hành động thực tế của agent.

B2Claude CodeからCodex CLIをMCPで呼ぶ——AGENTS.md単一ソース化で設定ファイルの分散を解決

📰 本文 + 和越対訳

Claude CodeとOpenAI Codex CLIの両方を契約しているエンジニアが直面する課題として、CLAUDE.mdとAGENTS.mdという2つの設定ファイルの乖離がある。シンボリックリンクは会社支給PCでは管理者権限が必要なため使えないことが多く、Gitでの同期もリポジトリをまたぐとリンクが壊れる問題がある。この記事では、~/.codex/に格納したAGENTS.mdを唯一の真実のソースとし、Claude側からは@import構文で参照するアーキテクチャを紹介している。具体的な手順として、まずclaude mcp add codex -- codex mcp-serverでCodexをMCPサーバーとして登録し、次にCLAUDE.mdから@import ~/.codex/AGENTS.mdの形で参照する。バージョン管理については、~/.codex/をGitリポジトリ化し、auth.jsonなどの認証情報は許可リスト方式の.gitignoreで保護することが強調されている。クロスプラットフォームなコードレビューワークフローを特権なしで実現できる実践的な提案だ。

Bài viết giải quyết vấn đề phổ biến mà các kỹ sư dùng cả Claude Code lẫn OpenAI Codex CLI gặp phải: duy trì hai file cấu hình CLAUDE.md và AGENTS.md đồng bộ với nhau. Symbolic link thường không dùng được trên PC do công ty cấp vì cần quyền admin, còn đồng bộ qua Git cũng gặp vấn đề khi link bị hỏng giữa các repository. Giải pháp là đặt AGENTS.md trong ~/.codex/ làm nguồn duy nhất, rồi tham chiếu từ CLAUDE.md bằng cú pháp @import. Các bước cụ thể: đăng ký Codex làm MCP server bằng claude mcp add codex -- codex mcp-server, sau đó dùng @import ~/.codex/AGENTS.md từ Claude. Về quản lý version, tác giả khuyến nghị đưa ~/.codex/ vào Git và bảo vệ file xác thực như auth.json bằng cách dùng allowlist .gitignore. Đây là giải pháp thực tiễn giúp thực hiện workflow review code xuyên nền tảng mà không cần quyền đặc biệt.

B3AIエージェントに実行させる前に——Sandbox・権限・ファイル境界の設計指針

📰 本文 + 和越対訳

Claude CodeやCodexといったAIコーディングエージェントを本番環境で安全に運用するためには、モデルの能力だけでなく、エージェントを囲む実行境界の設計が不可欠だ。この記事では、エージェントの実行境界を「ファイル境界」「コマンド境界」「ネットワーク境界」「認証境界」の4カテゴリに分類し、それぞれの設計指針を提示している。ファイル境界ではディレクトリへの読み書きアクセス制御とシークレットの保護、コマンド境界ではテストやパッケージマネージャーなど許可される操作の許可リストとデプロイ・DBマイグレーションなど制限される操作の明示化が重要だ。ネットワーク境界では外部アクセスの制限によりプロンプトインジェクションを防止し、認証境界ではエージェントへの認証情報漏洩を防ぐ。実装上の推奨として、調査フェーズは読み取り専用で開始し、特定ディレクトリへの書き込み権限は段階的に付与する方法が紹介されている。境界の内容はAGENTS.mdに文書化し、タスク仕様内に権限テーブルを設けることも提案されている。

Để vận hành an toàn các AI coding agent như Claude Code hay Codex trong môi trường thực tế, việc thiết kế ranh giới thực thi xung quanh agent quan trọng không kém gì năng lực của model. Bài viết phân loại ranh giới thực thi của agent thành 4 nhóm: ranh giới file (kiểm soát đọc/ghi thư mục, bảo vệ secret), ranh giới lệnh (allowlist các thao tác được phép như test, package manager; cấm deploy và DB migration), ranh giới mạng (hạn chế truy cập ngoài để ngăn prompt injection), và ranh giới xác thực (ngăn lộ credential cho agent). Về triển khai thực tế, tác giả khuyến nghị bắt đầu giai đoạn điều tra ở chế độ read-only, sau đó cấp quyền ghi cho từng thư mục cụ thể theo từng bước. Nội dung các ranh giới nên được ghi vào AGENTS.md và tạo bảng phân quyền trong đặc tả task, từ đó tạo vòng lặp cải thiện liên tục cho chính sách vận hành agent.

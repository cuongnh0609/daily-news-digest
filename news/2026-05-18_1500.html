B1コーディングエージェント時代にエンジニアは必要なのか——実験で見えた「読む力」の重要性

📰 本文 + 和越対訳

Zennに投稿されたこの記事では、著者がGWにClaude Code・Codex・GitHub Copilotといったコーディングエージェントを使ってSNSアプリをゼロから構築する実験を行い、今後のエンジニアの役割について真剣に考察した。著者の結論は「コードを書く人は大幅に減るが、エンジニアは消えない」だ。エージェントはAPIエンドポイントの追加や画面実装といった繰り返し作業は高速にこなすが、「何を作るべきか」を問い返す能力はない。そのため、エンジニアには問題定義・システム設計・生成コードのレビュー・タスク分解という4つの能力が今後ますます重要になる。特に「コードを読む力」の重要性が増しており、書くより読む・評価する能力が新しいコアスキルになると述べている。

Bài viết trên Zenn này ghi lại thí nghiệm của tác giả trong kỳ nghỉ Tuần lễ Vàng: sử dụng các coding agent như Claude Code, Codex, GitHub Copilot để xây dựng ứng dụng SNS từ đầu, qua đó đặt ra câu hỏi nghiêm túc về vai trò của kỹ sư phần mềm trong tương lai. Kết luận của tác giả là "số lượng người viết code sẽ giảm mạnh, nhưng kỹ sư sẽ không biến mất". Các agent hoàn thành nhanh chóng những công việc lặp đi lặp lại như thêm API endpoint hay implement màn hình, nhưng hoàn toàn không có khả năng đặt câu hỏi ngược lại "liệu có nên xây cái này không". Do đó, bốn năng lực ngày càng trở nên quan trọng với kỹ sư là: định nghĩa vấn đề, thiết kế hệ thống, review code được sinh ra, và phân tách task. Đặc biệt, "năng lực đọc code" đang trở nên quan trọng hơn viết code, và khả năng đánh giá, phê bình code sinh ra sẽ là core skill mới.

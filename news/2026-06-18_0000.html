B1Claude Code Hooks 入門 — 「お願い」を「必ず実行」に変える仕組み

📰 本文 + 和越対訳

Claude Code の Hooks は、CLAUDE.md のルールを LLM が「解釈して実行する」曖昧さを排除し、「条件が満たされれば必ず実行される」決定論的なメカニズムを提供する。従来の自然言語指示はモデルが状況に応じて判断するため、重要なセキュリティ制約やリント実行が漏れることがあった。Hooks はイベント駆動で動作し、PreToolUse・PostToolUse・セッション開始など30以上のイベントに対して command、http、mcp_tool、prompt、agent の5形式のアクションを設定できる。実用例として「rm -rf 等の危険コマンドのブロック」「コンテキスト自動注入」「編集後の自動リント」「commit 前の lint-staged 実行」「effort レベルの動的調整」「環境変数の自動セットアップ」などが挙げられる。設定は .claude/settings.json で定義し、バイパス権限モードでもセキュリティ強制が可能となるため、チーム開発での品質ゲート構築に特に有効だ。

Claude Code Hooks là cơ chế tất định (deterministic) giúp loại bỏ sự mơ hồ khi LLM "tự suy luận" để thực thi các quy tắc trong CLAUDE.md. Thay vì nhờ model tự phán đoán, Hooks đảm bảo "nếu điều kiện được thỏa mãn thì luôn luôn thực thi". Hooks hoạt động theo mô hình event-driven với hơn 30 sự kiện như PreToolUse, PostToolUse, session start, v.v., hỗ trợ 5 dạng action: command, http, mcp_tool, prompt, agent. Các use case thực tế bao gồm: chặn lệnh nguy hiểm như `rm -rf`, tự động inject context, chạy lint sau khi edit, thực thi lint-staged trước commit, điều chỉnh effort level động, và tự setup biến môi trường. Cấu hình được định nghĩa trong `.claude/settings.json` và đặc biệt hữu ích để xây dựng quality gate trong team development.

B2MCPとToolsをどう考えるか——read-only・write・権限スコープを分ける設計指針

📰 本文 + 和越対訳

AI エージェントに外部ツールを接続する際の安全設計について、「ツールは能力ではなく権限の束である」という視点から論じた記事が注目を集めている。著者は MCP（Model Context Protocol）および Tool の分類を3段階に整理することを提案する。第1段階は情報取得のみを行う Read-only tool（issue 読み取り、ログ検索など）、第2段階は作成・更新操作を行う Write tool（PR コメント作成、ステータス変更など）、第3段階は取り消しが困難な Destructive tool（デプロイ、マージ、本番 DB 更新など）だ。推奨事項として、Read-only から始めて段階的に権限を拡張すること、Write 操作は「下書き」フェーズで止めること、Destructive な操作は自動実行させないこと、そして外部ページの指示をエージェントが命令と誤認する「プロンプトインジェクション」対策を明示的に設けることが挙げられている。

Một bài viết đáng chú ý bàn về thiết kế an toàn khi kết nối công cụ bên ngoài cho AI agent, với quan điểm "tool là một tập hợp quyền truy cập, không chỉ là năng lực". Tác giả đề xuất phân loại MCP và Tool theo 3 cấp: (1) Read-only tool - chỉ đọc thông tin như issue, log; (2) Write tool - tạo/cập nhật như PR comment, thay đổi status; (3) Destructive tool - thao tác khó hoàn tác như deploy, merge, update production DB. Các khuyến nghị chính gồm: bắt đầu từ read-only và mở rộng quyền dần, dừng write operation ở bước "nháp", không cho agent tự động thực thi destructive operation, và thiết lập biện pháp chống prompt injection rõ ràng để agent không nhầm lệnh từ trang ngoài là chỉ thị hợp lệ.

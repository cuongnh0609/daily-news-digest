10How OpenAI Delivers Low-Latency Voice AI at Scale — Inside Their WebRTC Relay + Transceiver Architecture

📰 Article (English)

OpenAI published a detailed engineering post explaining the infrastructure behind ChatGPT Voice and the Realtime API's WebRTC endpoint, serving over 900 million weekly active users. The core architectural insight is a split between a stateless relay layer and a stateful transceiver: the relay is a lightweight UDP forwarder with a tiny public footprint, while the transceiver owns all WebRTC session state — ICE connectivity, DTLS handshake, SRTP keys, and session lifecycle. The first-packet routing problem is solved elegantly using the ICE ufrag (username fragment), which is already present in every STUN connectivity check. OpenAI encodes routing metadata inside the server-side ufrag so the relay can forward the very first packet to the correct transceiver without any hot-path lookup dependency. The Go implementation uses SO_REUSEPORT for multi-core UDP scaling, thread pinning via runtime.LockOSThread, and pre-allocated buffers to minimize GC pressure. A global relay fleet (powered by Cloudflare geo steering) shortens first-hop latency worldwide. The post also explains why they rejected SFU (selective forwarding unit) designs: most sessions are 1:1 and latency-critical, not multiparty, making the transceiver model a better fit.

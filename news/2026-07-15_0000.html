B-1Agent-as-a-Judgeは本当に必要なのか——評価精度90%の秘密は「計画」ではなく「証拠収集」

📰 本文 + 和越対訳

AIエージェントの評価手法として注目される「Agent-as-a-Judge（AaaJ）」について、その必要性と実態を掘り下げた考察記事が公開された。単純なLLM判定では最終出力しか見られないのに対し、複雑なエージェントタスクでは実行トレース・ファイルシステム・データベース上の副作用まで評価対象に含める必要がある。論文が示したAaaJの精度は人間専門家との一致率90%で、従来手法（70%）を大きく上回る。重要なのは、その精度向上はエージェント的な「計画」や「記憶」能力からではなく、グラフナビゲーション・ファイル読み取り・証拠の精密な参照という「環境アーティファクトへのアクセス」から生まれているという点だ。筆者は評価パラダイムを「証拠収集型（正解が環境上に存在するタスク向け）」と「視点分散型（主観評価に複数観点を導入）」の2つに整理する。AaaJは夢物語ではなく、実用的な必要性から生まれた手法であり、今後は障害診断や強化学習報酬信号への応用が見込まれる。

Một bài viết phân tích sâu về phương pháp "Agent-as-a-Judge (AaaJ)" — dùng AI agent để đánh giá các agent khác thay vì chỉ dùng LLM đơn giản. Với các tác vụ agent phức tạp, việc chỉ xét kết quả cuối cùng là không đủ; cần đánh giá cả execution trace, thay đổi trên filesystem, và tác động đến database. C-1Coding Agents Think 25 Steps Ahead: New Research Reveals Hidden "Latent Programming Horizons"

📰 Article (English)

A new paper from KTH Royal Institute of Technology — "Latent Programming Horizons in Coding Agents" (Silva, Tu, Monperrus; arXiv 2607.05188) — reveals something striking about how LLM-based coding agents work internally. Using linear probes on model hidden states, the researchers found that coding agents don't just react to the current state of the code; they actively encode future program properties in their activations well before taking action. Specifically, the models can accurately predict whether code will parse, pass tests, or reduce test failures approximately 25 editing steps ahead of the actual edits. These "latent programming horizons" represent a kind of internal lookahead that the model maintains without being explicitly prompted to plan. The probes trained on one coding benchmark transferred successfully to other benchmarks without retraining, suggesting that this predictive encoding is a genuine mechanistic capability rather than benchmark-specific overfitting. The authors note this has significant implications for agent design: if models already perform implicit planning in their latent space, explicitly forcing plan-then-act structures may be redundant or even counterproductive for some tasks. It also opens a direction for interpretability-driven agent monitoring — detecting when an agent "knows" a path will fail before it takes that path.

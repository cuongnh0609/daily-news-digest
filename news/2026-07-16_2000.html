A3AWS障害でPayPayなど広範なサービスが一時停止——CloudFrontのVPC Origins機能が原因

📰 本文 + 和越対訳

7月16日午後5時40分頃、AWSのCDNサービス「Amazon CloudFront」で大規模な障害が発生し、多数のウェブサービスへのアクセスが困難となった。直接的な影響を受けたのは、スマートフォン決済のPayPay、動画配信のニコニコ動画、ブログサービスのはてなブログ、コンテンツ投稿サービスのnoteなど、日本国内で広く利用されているプラットフォームだ。AWS公式の障害情報によると、CloudFrontの「VPC Origins」機能に起因するエラーが原因であり、この機能を利用しているサービスでパケットロスや接続タイムアウトが多発した。障害は断続的に継続し、各サービスはそれぞれ復旧作業を進めた。今回の障害は、多数のサービスが単一のクラウドインフラに依存していることのリスクを改めて浮き彫りにした。

Khoảng 17:40 ngày 16/7, dịch vụ CDN "Amazon CloudFront" của AWS gặp sự cố nghiêm trọng khiến nhiều dịch vụ web không thể truy cập được. Các dịch vụ bị ảnh hưởng trực tiếp bao gồm ví điện tử PayPay, nền tảng phát trực tuyến Niconico, dịch vụ blog Hatena Blog và nền tảng đăng nội dung note — tất cả đều được sử dụng rộng rãi tại Nhật Bản. Theo thông báo chính thức của AWS, nguyên nhân là lỗi trong tính năng "VPC Origins" của CloudFront, gây ra tình trạng mất gói tin và timeout kết nối. Sự cố này một lần nữa làm nổi bật rủi ro khi nhiều dịch vụ cùng phụ thuộc vào một hạ tầng cloud duy nhất.

7月末をもって旧来の健康保険証に関する暫定措置が終了し、8月からはマイナンバーカードを健康保険証として利用する「マイナ保険証」への移行が本格化する。暫定措置の終了後は、旧来の健康保険証を医療機関で提示した場合、医療費の全額（10割）を自己負担で求められる可能性があることが明らかとなった。この制度変更の影響を受けるのは約3700万人に上るとされており、対象者には速やかなマイナンバーカードの取得・健康保険証機能の登録が呼びかけられている。厚生労働省は医療機関側にも案内の強化を促しており、スムーズな移行に向けた対応を急いでいる。手続き方法についてはマイナポータルや市区町村の窓口で案内が受けられる。

📰 Article (English)

GitHub announced a significant security enhancement to Dependabot version updates: a mandatory default 3-day cooldown period before any new package version triggers an update pull request. This change directly addresses one of the most common vectors for software supply chain attacks — the window between a malicious package version being published and developers pulling it in via automated dependency updates.

The mechanics are straightforward: when a new package version appears on a registry, Dependabot waits 72 hours before creating a PR. Security updates are explicitly exempted from this cooldown, ensuring known CVEs can still be patched immediately. Teams can customize the cooldown duration via their .github/dependabot.yml configuration file — setting it higher for more caution or to zero to restore the original behavior. The feature is enabled by default for all repositories using Dependabot version updates as of the announcement date.

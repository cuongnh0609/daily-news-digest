B1Mastra Workspaces 発表——AIエージェントに「実際の作業環境」を与える4つの基盤コンポーネント

📰 本文 + 和越対訳

オープンソースのエージェントフレームワーク「Mastra」が、Workspacesと呼ばれる新機能群を発表した。これはAIエージェントを単なるテキスト生成ツールから、実際の業務を自律的に遂行できる「作業環境付き」のシステムへと昇格させることを目的としている。Workspacesは4つのコアコンポーネントから構成される。①Filesystem：サンドボックス化されたファイルアクセス権限。②Sandbox：安全なコマンド実行環境。③Skills：Agent Skills仕様に準拠した再利用可能なタスク定義。④Search：BM25キーワード検索とベクター意味検索のハイブリッド。セキュリティモデルが特徴的で、エージェントのアクセス権はプロンプトではなく設定レイヤーで制御される。`requireApproval`や`requireReadBeforeWrite`などのツール単位での権限付与が可能であり、エージェントが自分自身の権限を拡張することはできない。複数エージェントが同一のスキルセットを異なる権限ティアで共有できる設計も備える。ロードマップにはE2Bによるリモートサンドボックス、S3/GCSへのクラウドストレージ連携、マルチテナントSaaS向けスコープ付きワークスペースが含まれる。

Framework agent mã nguồn mở "Mastra" vừa công bố tính năng mới có tên Workspaces, nhằm nâng cấp AI agent từ công cụ tạo văn bản thành hệ thống tự vận hành có "môi trường làm việc" thực sự. Workspaces gồm 4 thành phần cốt lõi: Filesystem (quyền truy cập file trong sandbox), Sandbox (môi trường thực thi lệnh an toàn), Skills (định nghĩa task tái sử dụng theo chuẩn Agent Skills), và Search (tìm kiếm BM25 kết hợp vector semantic). Điểm đặc biệt là mô hình bảo mật: quyền truy cập của agent được kiểm soát ở tầng cấu hình, không phải qua prompt, với các tham số như `requireApproval` và `requireReadBeforeWrite` cho từng tool. Agent không thể tự mở rộng quyền của mình. Nhiều agent có thể dùng chung một bộ skills với các cấp quyền khác nhau. Roadmap bao gồm remote sandbox qua E2B, tích hợp cloud storage (S3/GCS), và scoped workspace cho multi-tenant SaaS.

B2RAGのコストを1/15に削る4層アーキテクチャ——「毎回LLMを呼ばない」設計の実装論

📰 本文 + 和越対訳

静的コーパスを扱うRAGシステムにおいて、月間100万クエリ規模で15〜20倍のコスト削減を実現する4層アーキテクチャの手法が公開された。核心にあるのは「事前計算」の発想だ。各ドキュメントチャンクに対し、前処理の段階で5〜10個の想定質問とその回答を生成してキャッシュしておく。クエリ時には、事前生成した質問群との意味的類似度を計算し、高類似度のものはキャッシュ済み回答を直接返すことでLLM呼び出しをゼロにする。Layer 0はそもそも検索が不要なクエリを40〜60%フィルタリングする役割を担う。Layer 2はHaikuと prompt cachingを組み合わせた通常RAGとして機能し、Layer 3の高コストモデル（Sonnet級）はトラフィック全体の5〜15%の複雑なクエリにのみ使用される。筆者は本番環境での6つの落とし穴として、ベクターDB選定、埋め込みの一貫性、バッチ処理の信頼性、LLMトークン以外のインフラコスト、レイテンシの積み上がり、同時実行制限を挙げている。

Một kiến trúc RAG 4 tầng có thể giảm chi phí 15-20 lần ở quy mô 1 triệu query/tháng với corpus tĩnh đã được chia sẻ. Ý tưởng cốt lõi là "tính toán trước": với mỗi chunk tài liệu, sinh sẵn 5-10 câu hỏi dự kiến và câu trả lời tương ứng, lưu vào cache ngay từ bước tiền xử lý. Khi có query, hệ thống tính độ tương đồng ngữ nghĩa với các câu hỏi đã sinh sẵn; nếu độ tương đồng cao thì trả về câu trả lời từ cache mà không gọi LLM. Layer 0 lọc 40-60% query không cần tìm kiếm. Layer 2 dùng RAG thông thường với Haiku + prompt caching. Layer 3 chỉ dành cho 5-15% query phức tạp với model đắt tiền hơn (Sonnet). Tác giả liệt kê 6 vấn đề thường gặp trong môi trường production: chọn vector DB, tính nhất quán embedding, độ tin cậy batch processing, chi phí infrastructure ngoài LLM token, độ trễ tích lũy, và giới hạn concurrency.

The key requirement: AI-generated bug reports must include a reproducible test case plus a demonstration patch showing one possible fix. Lead maintainer D. As coding agents become capable of submitting PRs autonomously, projects need explicit policy documents to communicate contribution boundaries. SQLite's AGENTS.md sets a clear precedent: welcome AI as a bug-finder, reject AI as a code author. The "demonstration patch" requirement is particularly clever — it forces enough specificity to separate quality signal from noise without accepting the code directly.

C2Microsoft Copilot Cowork Had a File Exfiltration Vulnerability — Prompt Injection Chains Email + Image + OneDrive

📰 Article (English)

A significant security vulnerability was discovered and disclosed in Microsoft's Copilot Cowork product that allowed unauthorized file exfiltration. The attack exploited a chain of individually reasonable design decisions that combined into a critical flaw. The exploit chain worked as follows: Copilot Cowork agents could send emails to users' inboxes without requiring explicit approval. These emails could embed external images that, when rendered by the email client, triggered outbound network requests to attacker-controlled servers. When this capability was combined with OneDrive's pre-authenticated download links, a prompt injection attack became dangerous: an attacker could craft malicious content that caused the agent to generate shareable OneDrive file links inside an email message. When the victim opened the email, the image render request leaked those OneDrive links to the attacker's server — granting unauthorized access to the victim's files. The critical lesson here extends beyond this specific product: agentic systems create compound attack surfaces. Each individual feature (agent email sending, image rendering, link generation) may seem safe in isolation, but when combined without sufficient approval gates and permission boundaries, they create exploitable chains. This is why agent security frameworks need to treat outbound actions — especially those involving user data — as requiring explicit human approval, not just prompt-level instructions. Microsoft has since addressed the vulnerability, but the incident underscores a fundamental challenge in productizing agentic AI.

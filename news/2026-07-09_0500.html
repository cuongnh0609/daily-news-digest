A3ギガファイル便、長期障害からほぼ復旧——ドメイン変更で対応

📰 本文 + 和越対訳

無料の大容量ファイル転送サービス「ギガファイル便」は7月8日、6月下旬から続いていたシステム障害からほぼ復旧したと発表した。同社は復旧のための措置として、従来のドメイン「gigafile.nu」から「gigafile.jp」への一時的な移行を行い、サービス提供を再開した。障害期間中にアクセスされたファイルのURLが機能しない問題については、新ドメインからの再アクセスが可能になる予定とされている。障害は6月下旬午前9時ごろに発生し、約2週間にわたりサービスが不安定な状態が続いた。ギガファイル便は個人・法人問わず広く利用されているサービスであり、今回の長期障害はSNSで話題を集め、代替サービスへの移行を検討するユーザーも相次いだ。今後はドメイン変更を経ながら通常通りの利用が可能になるとしている。

Dịch vụ truyền tải file dung lượng lớn miễn phí "GigaFile" ngày 8/7 thông báo đã gần như khôi phục hoàn toàn sau sự cố hệ thống kéo dài từ cuối tháng 6. Để khôi phục dịch vụ, công ty đã tạm thời chuyển từ domain "gigafile.nu" sang "gigafile.jp". Các URL file được truy cập trong thời gian sự cố không hoạt động dự kiến sẽ có thể truy cập lại qua domain mới. Sự cố bắt đầu khoảng 9 giờ sáng cuối tháng 6 và khiến dịch vụ không ổn định suốt gần 2 tuần. GigaFile là dịch vụ được sử dụng rộng rãi bởi cả cá nhân lẫn doanh nghiệp, và sự cố kéo dài này đã thu hút nhiều chú ý trên mạng xã hội, khiến không ít người dùng tìm kiếm dịch vụ thay thế.

B1Claude Codeで作ったアプリ、3ヶ月後の現実——AIコードの「敵対的レビュー」が必須と判明

📰 本文 + 和越対訳

Zennのエンジニア柏岡氏が、Claude Codeを使って開発したMarkdown閲覧専用デスクトップアプリ「Rendu」の3ヶ月後の現状を報告した。リリース数が3から15へ増加し、コミット数も112から199に伸び、海外からの貢献によって中国語ローカライズも実現するなど、プロジェクト自体は順調に成長した。しかし最大の発見は、「AIが生成したコードをAIに監査させたところ、実行可能な脆弱性が見つかった」という事実だ。信頼できない.mdファイル内のリンクをクリックするだけで、パス操作による任意コード実行が可能になる脆弱性が検出されたという。著者はこの経験から、AIによるコード生成のコストがほぼゼロになった時代こそ、検証と責任の重要性がかえって高まると警鐘を鳴らしている。「実コード照合」「敵対的レビュー」「実機検証」の多段階検証アプローチが今後の開発で不可欠になると結論付けた。

Kỹ sư Kashioka trên Zenn đã báo cáo về tình trạng của ứng dụng desktop chỉ đọc Markdown "Rendu" được xây dựng bằng Claude Code sau 3 tháng. Dự án phát triển thuận lợi với số lần release tăng từ 3 lên 15, commit từ 112 lên 199, và thậm chí có người đóng góp từ nước ngoài để thêm bản địa hóa tiếng Trung. Tuy nhiên, phát hiện quan trọng nhất là "khi dùng AI để kiểm tra code do AI tạo ra, đã phát hiện một lỗ hổng bảo mật có thể khai thác được". Cụ thể, chỉ cần click vào một liên kết trong file `.md` không đáng tin cậy có thể dẫn đến thực thi code tùy ý thông qua tấn công path traversal. 📰 Article (English)

Microsoft has released TypeScript 7.0, the most significant rewrite of the compiler in the project's history. The new version ports the entire TypeScript compiler and type checker from JavaScript to Go, delivering build speed improvements of 8x to 12x on real-world projects. The VS Code codebase, used as a benchmark, saw its build time drop from 125.7 seconds down to just 10.6 seconds. The rewrite introduces true multithreading via Go's goroutines, a rebuilt watch mode powered by the Parcel watcher, and new parallelization flags — --checkers and --builders — for fine-grained control. The language server crash rate dropped by over 60%, substantially improving IDE reliability for all editor integrations. Breaking changes are significant: ES5 targeting and legacy module modes (commonjs, amd) have been removed, strict mode defaults are now enforced, and rootDir and types must be explicitly configured in non-standard setups. Embedded language tooling (Vue, Angular, Svelte) must stay on TypeScript 6.0 for now, as TypeScript 7.0 support for those ecosystems is still in progress. A compatibility package is available for gradual migration. Migration from TypeScript 6 requires updating tsconfig.json and resolving removed legacy targets before upgrading.

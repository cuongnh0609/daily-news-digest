B1Claude Codeの.claude/ディレクトリ「賞味期限」問題——半年運用して判明した自己改善スキルループの設計

📰 本文 + 和越対訳

Claude Codeを本番プロジェクトで運用していると、.claude/ディレクトリ内に蓄積されたスキルやコンテキストファイルが 時間の経過とともに「腐敗」していく問題が顕在化する。 著者はこの問題を半年以上の実運用を通じて観察し、スキルの「賞味期限切れ」現象と命名した。 具体的には、プロジェクトのコードベースが進化するにつれて古いスキルが誤った実装を誘導したり、 廃止されたパターンを参照し続けたりするケースが頻発したという。 解決策として提案されたのは、自己改善スキルループの構築—— 定期的にスキルの内容をレビューするタスクをClaude Code自身に割り当て、 プロジェクトの現状と乖離が生じたスキルを自動更新する仕組みだ。 このアーキテクチャによって、スキルの陳腐化を防ぎながら継続的に品質を維持できることが示されている。

Khi vận hành Claude Code trong dự án thực tế, vấn đề các file skill và context tích lũy trong thư mục .claude/ dần bị "hỏng" theo thời gian trở nên rõ rệt. Tác giả đã quan sát hiện tượng này qua hơn nửa năm vận hành thực tế và đặt tên là hiện tượng "hết hạn sử dụng" của skill. Cụ thể, khi codebase của dự án phát triển, các skill cũ dẫn dắt việc implement sai hoặc liên tục tham chiếu các pattern đã bị loại bỏ xảy ra thường xuyên. Giải pháp được đề xuất là xây dựng vòng lặp skill tự cải thiện — giao cho chính Claude Code nhiệm vụ định kỳ review nội dung skill, và tự động cập nhật các skill bị lệch so với thực trạng dự án. The Zig programming language team has published a major architectural rework of the Zig build system, one of the most discussed changes in the language's evolving ecosystem. The rework decouples the compiler's incremental cache from the build graph execution layer, enabling significantly faster incremental builds and more predictable caching behavior. Previously, the build system and compiler cache were intertwined in ways that caused subtle cache invalidation bugs and made distributed builds difficult to implement correctly. The change is part of Zig's ongoing effort to become a serious contender for systems programming alongside C and Rust.

B1Claude Codeのsettings.json階層を理解する——user/project/localの優先順位と最小設定の実践

📰 本文 + 和越対訳

Claude Codeにはsettings.jsonによる細かい設定管理の仕組みがあり、スコープが「user（グローバル）」「project（リポジトリ単位）」「local（端末ローカル）」の3層に分かれている。優先順位はlocal → project → userの順で、より近いスコープの設定が上書きする形になる。記事では、各スコープのファイルパスと実際にどんな設定項目（モデル選択・ツール許可・カスタム指示など）が有効かを整理している。また最小限の設定で快適に使い始めるためのスターターテンプレートも紹介されており、チーム開発でのproject/settings.json共有運用についても言及がある。Claude Codeを使い込む上で押さえておきたい基礎知識として注目されている。

Claude Code có cơ chế quản lý cài đặt thông qua settings.json với 3 lớp phạm vi: "user (global)", "project (theo repository)" và "local (máy cục bộ)". Thứ tự ưu tiên là local → project → user, các cài đặt có phạm vi gần hơn sẽ ghi đè lên. Bài viết tổng hợp đường dẫn file của từng phạm vi và các mục cài đặt thực tế (chọn model, cấp phép tool, hướng dẫn tùy chỉnh...). Tác giả cũng giới thiệu starter template tối giản để bắt đầu sử dụng nhanh, và đề cập đến cách chia sẻ project/settings.json trong phát triển nhóm. Đây là kiến thức nền tảng không thể thiếu khi sử dụng Claude Code chuyên sâu.

B2AWS Transform – Continuous Modernizationプレビュー——AIエージェントがリポジトリを継続スキャン、技術的負債を自動検出・修正

📰 本文 + 和越対訳

Amazon Web Services（AWS）は、AIエージェントがコードリポジトリを継続的に自動スキャンし、技術的負債を検出・優先順位付けして報告・修正まで支援する新サービス「AWS Transform – Continuous Modernization」のプレビューを公開した。サポート終了（EOL）したライブラリやフレームワークの使用、セキュリティ上の問題、コードの非推奨パターンなどを検出対象としており、検出結果はプルリクエストとして提案される。SonarQubeやSnykなど既存の静的解析ツールとの差別化として、コンテキストを理解した上での修正提案が強みとされる。エンタープライズ向けの大規模レガシーコードベース近代化を想定したサービスだ。

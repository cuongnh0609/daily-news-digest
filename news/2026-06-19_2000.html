📰 本文 + 和越対訳

LayerXのエンジニアによるZenn記事が注目を集めている。AIエージェントが自らのワークフローを改善する際、なぜ90%程度の精度で頭打ちになり、初期設計の前提を根本から見直せないのかを分析した内容だ。記事では三つの根本的な問題を指摘している。初期要件への固執（アンカリングバイアス）、自己振り返りだけでは発散的思考が生まれないこと、そして探索空間が狭くなりがちなことだ。PACEvolveやCSEなどの研究を引用しつつ、解決策として「複数の候補解を並行して管理し、過去の試みを記録して多様性指標を評価基準に加える」ことを提案している。結論として、モデルの能力向上よりもハーネス——エージェントの探索を導くインフラ設計——の改善がブレークスルーの鍵だと強調している。

C-01GitHub Actions Closes "Pwn Request" Gap — Safer pull_request_target Defaults Ship in actions/checkout v7

📰 Article (English)

GitHub shipped a critical security fix in actions/checkout v7 that closes the long-standing "pwn request" attack vector in GitHub Actions. The vulnerability exploits the pull_request_target event, which grants workflows access to base repository secrets and tokens. When an unreviewed fork's code is checked out under this event, an attacker can run malicious code with full workflow privileges — a common supply-chain risk pattern that has affected major open-source projects. The new safeguard automatically refuses to check out fork pull request heads or their merge commits when triggered by pull_request_target or workflow_run events with risky configurations. Developers who legitimately need this behavior can opt out using the new allow-unsafe-pr-checkout input flag, which forces an explicit security decision. Starting July 16, 2026, GitHub will backport these protections to all supported major versions. Workflows pinned to floating tags like @v4 will receive the update automatically; those pinned to specific SHA or version numbers require a manual upgrade. GitHub recommends all teams audit their pull_request_target usage before the backport date to avoid unexpected failures.

C-03"Are You in the Weights?" — New Tool Lets You Check if Your Writing Appears in LLM Training Data

📰 Article (English)

A new web tool called "Are You in the Weights?" launched on Hacker News front page on June 18, quickly accumulating 356 points and significant discussion. The service allows individuals to probe whether their writing, code, or other personal content may have been incorporated into the training data of frontier AI models including GPT-5.5, Gemini Ultra, Claude Sonnet, and several smaller open-weights models. The tool takes a text sample provided by the user and runs membership inference tests across multiple models, returning a probability estimate of whether that specific content was part of each model's training corpus. Unlike prior art — which typically required direct model access and technical expertise — this tool exposes the capability through a simple web interface accessible to non-researchers. The launch sparked debate on Hacker News around several dimensions: the reliability of membership inference at scale, privacy implications for content creators and open-source developers whose code may be in training data without consent, and the practical implications for copyright litigation currently underway against major AI labs. The Hacker News thread noted that false positives are a concern but that the tool's statistical framing makes the caveats reasonably clear to informed users.

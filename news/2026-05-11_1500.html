B1Claude CodeにAWSコスト調査をさせたら数千万円分の改善ポイントが見つかった——GENIEEのSREが実践報告

📰 本文 + 和越対訳

広告テクノロジー企業GENIEEのSREチームが、Claude Codeを活用してAWS環境全体のコスト最適化調査を実施した事例を公開した。調査はClaude Codeに読み取り専用のAWS CLIプロンプトを与えるだけという最小構成で開始され、Claude Codeは自律的にCost Explorerを参照してコスト異常を検出し、その原因をソースコードまで遡って特定した。発見された改善案は年間数千万円規模に上るとされ、その内容も具体的だった。第一の事例はLambda関数がAPIリクエストごとにS3へのPutObjectを呼び出していたものの、そのログはCloudWatch Logsと完全に重複しており一切参照されていないというものだった。第二の事例はAPI GatewayのレスポンスデータがJSON非圧縮のまま送出されていたケースで、Serverless Frameworkの設定1行を追加するだけで転送量が86.4%削減できると示した。著者は「修正そのものは単純。難しいのは問題の存在を知ることだった」と述べており、長期運用で蓄積した非効率をAIが先入観なくスキャンできる強みを評価している。

Đội ngũ SRE của công ty công nghệ quảng cáo GENIEE đã chia sẻ kết quả thực tế khi sử dụng Claude Code để điều tra tối ưu hóa chi phí toàn bộ môi trường AWS. Cấu hình tối giản chỉ cần cấp quyền đọc AWS CLI cho Claude Code — công cụ tự động tham chiếu Cost Explorer, phát hiện bất thường và truy ngược đến mã nguồn. Các cải tiến được tìm thấy ước tính tiết kiệm hàng chục triệu yên mỗi năm. Trường hợp đầu tiên: hàm Lambda gọi PutObject lên S3 mỗi lần có API request, nhưng log đó hoàn toàn trùng lặp với CloudWatch Logs và không bao giờ được đọc lại. Trường hợp thứ hai: response từ API Gateway không được nén, và chỉ cần thêm một dòng vào cấu hình Serverless Framework là giảm được 86,4% dung lượng truyền tải. Tác giả nhận xét: "Bản thân việc sửa rất đơn giản. Điều khó là biết rằng vấn đề đó tồn tại" — đánh giá cao khả năng AI quét toàn bộ hệ thống mà không có định kiến, phát hiện những inefficiency tích lũy qua nhiều năm vận hành.

B2ReDel再現実装：LLMが自ら委譲する再帰マルチエージェントの仕組みを解説

📰 本文 + 和越対訳

EMNLP 2024で発表されたマルチエージェントフレームワーク「ReDel（Recursive Delegation）」の再現実装を通じて、LLMが動的にサブエージェントを生成・委譲する仕組みを解説した記事がZennで公開された。ReDel最大の特徴は、委譲そのものをLLMが呼び出せるツールとして実装している点だ。従来のLangGraphやCrewAIが静的なグラフ定義に頼るのとは異なり、ReDel ではエージェントが必要に応じて「DelegateOne」や「DelegateWait」関数を呼び出し、実行時に動的にサブエージェントを生成する。この構造により、タスク分解の戦略そのものをLLMに委ねられるため、設計者があらかじめすべての分岐を定義する必要がなくなる。また、親タスクをそのまま子エージェントに渡す「過剰委譲」を防ぐため、rapidfuzzによるタイトル類似度（閾値80%）チェックを採用している。FanOutQAやTravelPlannerの2ベンチマークで当時のSOTAを上回る結果を記録しており、再帰的なエージェント設計に関心のある開発者に必読の内容となっている。

Một bài viết được đăng trên Zenn giải thích cơ chế tái hiện ReDel (Recursive Delegation) — framework đa agent được trình bày tại EMNLP 2024. Điểm đặc biệt nhất của ReDel là việc ủy quyền (delegation) được triển khai như một công cụ mà LLM có thể gọi trực tiếp. Khác với LangGraph hay CrewAI dựa trên định nghĩa đồ thị tĩnh, ReDel cho phép agent gọi hàm `DelegateOne` hoặc `DelegateWait` để tự động sinh sub-agent tại thời điểm chạy (runtime). Nhờ đó, chiến lược phân rã nhiệm vụ được giao phó hoàn toàn cho LLM — người thiết kế không cần định nghĩa trước tất cả các nhánh. Để tránh "ủy quyền thừa" (giao nguyên xi nhiệm vụ gốc cho agent con), hệ thống áp dụng kiểm tra độ tương đồng tiêu đề qua rapidfuzz (ngưỡng 80%). ReDel đã vượt SOTA trên 2 trong 3 benchmark (FanOutQA, TravelPlanner), là tài liệu đọc bắt buộc với developer quan tâm đến agentic system design.

📰 本文 + 和越対訳

LLMが日本の法律・税務に関する質問に矛盾する回答を返す問題を解決するため、エンジニアのshujiさんがMCPサーバー群「法規MCPファミリー」を開発・公開した。中核となるのは4つのコンポーネントだ。①「houki-egov-mcp」はe-Gov API v2を通じて法令・省令・改正履歴を取得し、略称を正式名称に解決する機能を持つ。②「houki-nta-mcp」はSQLite＋全文検索（FTS5）で国税庁のQ&Aや基本通達を検索可能にし、回答の法的拘束力を示す「legal_status」メタデータを付与する。③「houki-abbreviations」は「消基通」→「消費税法基本通達」のような略称辞典を共有ライブラリとして提供する。④「houki-research-skill」はMCPではなくClaudeのSkillとして実装し、税理士法52条・弁護士法72条の専門家資格制限や定型的な引用形式を強制する行動ルールを管理する。設計の肝は「データ取得層（MCP）」と「行動制御層（Skill）」を分離した点にあり、MCPが他の用途に流用されても制約が混入しない構造を実現している。

Kỹ sư shuji đã phát triển và công bố bộ MCP server "Houki MCP Family" nhằm giải quyết vấn đề LLM đưa ra câu trả lời mâu thuẫn khi được hỏi về luật pháp và thuế của Nhật. Bộ công cụ gồm 4 thành phần: ① `houki-egov-mcp` truy cập e-Gov API v2 để lấy luật, nghị định, lịch sử sửa đổi và giải quyết tên viết tắt; ② `houki-nta-mcp` tìm kiếm toàn văn Q&A và thông tư cơ bản của Cục Thuế Quốc gia qua SQLite + FTS5, gắn metadata `legal_status` cho biết mức độ ràng buộc pháp lý; ③ `houki-abbreviations` cung cấp từ điển chung giải quyết các từ viết tắt pháp lý; ④ `houki-research-skill` được triển khai như Claude Skill (không phải MCP) để quản lý các quy tắc hành vi — bao gồm giới hạn tư vấn chuyên nghiệp theo điều 52 Luật Thuế và điều 72 Luật Luật sư. Điểm cốt lõi của thiết kế là tách biệt "lớp lấy dữ liệu (MCP)" và "lớp kiểm soát hành vi (Skill)", đảm bảo các ràng buộc không bị lẫn vào khi MCP được tái sử dụng cho mục đích khác.

C1AI-Generated "Garbage Reports" Flood Open Source Bug Bounties — HackerOne Halts Program

📰 Article (English)

HackerOne has suspended its open source bug bounty program — originally launched in 2021 and having distributed over $15 million in rewards — due to an overwhelming surge of AI-generated vulnerability reports that are predominantly false positives. The policy change, enacted on March 28, now classifies AI-sourced submissions as "garbage reports" ineligible for compensation. The problem is not isolated to HackerOne: the Node.js project has also halted its bug bounty program and is rejecting AI-generated reports outright. The curl project has formally excluded such submissions from its scope. Google updated its Open Source Security Vulnerability Reward Program to label AI-sourced reports as ineligible. Security experts note that AI tools can generate high volumes of plausible-sounding but technically invalid vulnerability reports at a rate no human team can meaningfully triage. The flood shifts burden to maintainers, making it harder to distinguish genuine disclosures from noise — and dramatically extends remediation timelines for real issues. This represents a new class of attack surface: not against systems, but against the human processes that defend them.

C2The Unreasonable Effectiveness of HTML: Claude Code's Output Format That Changes Everything

📰 Article (English)

Simon Willison highlights a post by Thariq Shihipar from Anthropic's Claude Code team making a compelling case for a simple but underused practice: requesting HTML output instead of Markdown when working with Claude. The argument is that HTML unlocks capabilities unavailable in plain text — inline SVG diagrams, interactive JavaScript widgets, in-page navigation, rich styled layouts, and embedded tables. Historically, developers defaulted to Markdown due to the tight context window constraints of GPT-4 era models (8,192 tokens), where generating rich HTML felt wasteful. That tradeoff is now obsolete: modern Claude models operate with far larger context budgets, making HTML the clearly superior output format for complex explanations, documentation, data presentations, and technical walkthroughs. The practical technique is to prompt Claude with something like "Explain this code in detail, output as a styled, interactive HTML/CSS/JavaScript page." The result is a self-contained artifact that can be opened in any browser, shared, and navigated — vastly more useful than a wall of Markdown. Willison demonstrated this by having Claude produce an interactive HTML explanation of a Linux security exploit, complete with styled callouts, tables, and navigation links.

C3You Need an AI Coding Agent That Reduces Your Maintenance Costs, Not Increases Them

📰 Article (English)

James Shore, author of "The Art of Agile Development," challenges a popular assumption about AI coding agents in a new post on his blog. While AI tools are celebrated for dramatically accelerating code generation speed, Shore argues that the dominant metric being optimized — lines of code per hour — is the wrong one. The actual cost of software is not writing code but maintaining it over months and years. An AI agent that generates verbose, poorly-structured, or under-tested code at high speed may reduce time-to-first-commit but substantially increase total maintenance burden. Shore presents a framework for evaluating AI coding agents based on: (1) does it reduce technical debt over time or accumulate it, (2) does generated code pass rigorous automated testing or merely appear to work, (3) does it improve or degrade code readability for the next developer, and (4) does it make the codebase easier or harder to change in the future. The post is a direct counter to "vibe coding" culture — writing code fast without regard for structure or testability. Shore argues that the winning AI-augmented developer is not the fastest code generator, but the one who uses AI to build software that is cheaper to own over its lifetime.

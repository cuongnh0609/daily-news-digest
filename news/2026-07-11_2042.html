C-03GitHub Enterprise Now Enforces Centralized OpenTelemetry Export for Copilot in VS Code and CLI

📰 Article (English)

GitHub has released enterprise-managed OpenTelemetry (OTel) export for both the Copilot Chat extension in VS Code and the Copilot CLI agent host process. Previously, individual developers had to manually configure OTLP environment variables to route telemetry data to an organization's observability backend. With this update, IT administrators can push a centralized OTel configuration through three delivery channels: native MDM (Windows Registry or macOS managed preferences), server-managed settings from the GitHub account, or a file-based managed-settings.json. The managed settings control the OTLP endpoint, transport protocol (otlp-http or otlp-grpc), OTel service name, resource attributes, exporter headers (including authentication tokens), and content capture policies for prompts, responses, and tool call data. A critical security detail: managed exporter headers are applied exclusively to the Copilot Chat OTLP exporter and are never propagated as environment variables, preventing authentication tokens from leaking to subprocess tools. This feature is available to GitHub Copilot Enterprise customers and significantly reduces per-developer configuration overhead while enabling audit and compliance workflows at scale.

C-02npm v12 Breaking Changes: Script Execution, Git Deps, and Remote URLs Now Require Explicit Approval

📰 Article (English)

npm v12, scheduled for release in July 2026, introduces three security-focused behavioral changes that will break existing workflows for many Node.js developers. First, dependency install scripts (the postinstall, preinstall hooks in third-party packages) will no longer execute automatically — developers must explicitly run npm approve-scripts to audit and approve which packages are allowed to execute scripts. Second, Git dependencies (packages installed directly from a GitHub URL or git repository) will require an explicit opt-in flag, as they bypass the npm registry's security review process. Third, remote URL installations will similarly require confirmation. These three behaviors were previously opt-out; npm v12 flips them to opt-in security controls. Teams can prepare today by upgrading to npm 11.16.0 or newer, which already emits warnings for all three cases, giving development teams time to audit their dependency scripts before the breaking change lands. The changelog notes this is part of a broader npm security hardening initiative following several supply chain attacks that exploited install scripts.

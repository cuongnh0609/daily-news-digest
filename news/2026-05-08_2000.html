C3Mozilla Proposes WAICT: Cryptographic Code Integrity for the Open Web Against Malicious Server Injection

📰 Article (English)

Mozilla's Firefox Security Team published a proposal called WAICT (Web Application Integrity, Consistency and Transparency) — a new web security architecture that cryptographically binds client-side JavaScript to a publicly auditable transparency log, similar in concept to Certificate Transparency for TLS. The core problem WAICT solves: today, any server can silently serve different JavaScript to different users (for targeted attacks, A/B testing, or coercion by authorities), and the browser has no way to detect it. WAICT introduces a commitment scheme where web apps register a Merkle-tree root hash of their JavaScript bundle to a public log. On every page load, the browser verifies the served script against the log entry; if the script deviates, the browser raises a security warning. The proposal also covers service worker integrity, preventing compromised service workers from modifying page code silently. This is particularly relevant to high-stakes web apps — banking, e-government, secure messaging — where users need assurance that the code running in their browser matches the audited version. Mozilla is proposing WAICT as a new W3C standard, and the Firefox Security Team is seeking feedback from the web community. It builds on lessons from Certificate Transparency and is designed to be deployable without requiring changes to existing CDN infrastructure.

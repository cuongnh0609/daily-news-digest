B1Claude Code vs Codex——iOSリファクタリングで見えた「暗黙の開発者要件」の違い

📰 本文 + 和越対訳

料理動画サービスkurashiruを運営するdely社のエンジニアが、大規模iOSリファクタリングを通じてClaude Code（Opus 4.7）とCodex（GPT-5.5）を実際に比較した記事が注目を集めている。両ツールの本質的な違いは「探索戦略」にあり、Claude Codeは深さ優先型で仮説を立てて一気に実行する傾向があるのに対し、Codexは幅優先型でエッジケースやトレードオフを丁寧に調べながら進む。この違いは開発者に求める暗黙の役割にも影響する。Claude Codeには事前に十分な文脈設計と明確な指示が必要で、Codexには実装中の継続的な関与が求められる。著者は最終的に「調査フェーズをCodexで行い、実装フェーズをClaude Codeで実行する」という組み合わせが最も効果的だと結論づけており、エージェント駆動開発における役割分担の重要性を示している。

Một kỹ sư tại dely (công ty vận hành dịch vụ video nấu ăn kurashiru) đã so sánh thực tiễn Claude Code (Opus 4.7) và Codex (GPT-5.5) trong dự án refactor iOS quy mô lớn. Sự khác biệt cốt lõi nằm ở "chiến lược khám phá": Claude Code hoạt động kiểu depth-first — lập giả thuyết và thực thi quyết đoán — trong khi Codex hoạt động kiểu breadth-first, tỉ mỉ khảo sát các edge case và đánh đổi. Điều này ảnh hưởng đến vai trò ngầm định mà mỗi công cụ đòi hỏi từ developer: Claude Code cần được thiết kế context rõ ràng trước khi chạy, còn Codex cần developer duy trì sự tham gia liên tục trong suốt quá trình. Tác giả kết luận rằng kết hợp hiệu quả nhất là "dùng Codex cho giai đoạn điều tra, Claude Code cho giai đoạn thực thi", cho thấy tầm quan trọng của việc phân công vai trò trong agentic-driven development.

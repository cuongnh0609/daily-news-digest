C1Zed Introduces DeltaDB — A Version Control System Built for AI-Collaborative Software Development

📰 Article (English)

Zed has announced DeltaDB, a new version control system designed to replace Git's commit-centric model with one that captures every individual operation between snapshots. The core insight driving the design is that modern software development — especially with AI agents in the loop — increasingly happens "between commits," in the conversations, decisions, and incremental changes that traditional VCS tools discard. DeltaDB tracks fine-grained deltas with stable identities that survive code refactoring, supports conflict-free replicated worktrees enabling true simultaneous multi-user editing without merge conflicts, and introduces anchored references that bind discussions directly to evolving code — not brittle line numbers that shift with every edit. For AI-assisted workflows, DeltaDB gives agents a structured record of the context behind every change: why a function was introduced, which discussion led to a refactor, and what the code looked like at each decision point. The result is a collaboration substrate where human developers and AI agents share the same continuous, semantically rich history rather than communicating only through atomic commits.

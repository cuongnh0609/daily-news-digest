B-01GPT-5.5 向け「指示の型」設計――3モード Agent Skill とコスト最適ルーティング戦略

📰 本文 + 和越対訳

OpenAIのGPT-5.5は高い推論能力を持つ一方、推論トークンのコストが大きく、従来の詳細な手順書型プロンプトよりも「成果優先のブリーフ形式」が効果的だとされる。Zennユーザーのshinyay氏はGitHub Copilot Agent Skillとして、GPT-5.5向けのプロンプト最適化フレームワークを公開した。このスキルはGenerate（タスクからブリーフを自動生成）・Audit（既存プロンプトの28カテゴリ反パターン検査）・Route（コスト意識のモデル振り分け）という3つのモードで動作する。とりわけ注目されるのはRouteモードで、まずgpt-5.4-miniや5.4など安価なモデルで処理し、検証失敗・セキュリティ懸念・アーキテクチャの不確実性といったトリガーが発火した場合のみGPT-5.5にエスカレーションする仕組みだ。33のリファレンス文書とCLIツールが付属し、オフライン利用にも対応している。

GPT-5.5 của OpenAI có khả năng suy luận mạnh mẽ nhưng chi phí reasoning token rất cao, do đó "brief tập trung vào kết quả đầu ra" hiệu quả hơn kiểu prompt liệt kê các bước chi tiết truyền thống. Người dùng Zenn shinyay đã công bố một GitHub Copilot Agent Skill là framework tối ưu hóa prompt cho GPT-5.5 với 3 chế độ: Generate (tự động tạo brief từ mô tả task), Audit (kiểm tra prompt theo 28 danh mục anti-pattern), và Route (phân phối model theo chi phí). Đáng chú ý nhất là chế độ Route — hệ thống xử lý trước bằng các model rẻ hơn như gpt-5.4-mini, chỉ escalate lên GPT-5.5 khi có trigger như xác thực thất bại, lo ngại bảo mật, hoặc kiến trúc không chắc chắn. Skill đi kèm 33 tài liệu tham khảo và công cụ CLI, hỗ trợ dùng offline.

C2PyPI Now Rejects File Uploads to Releases Older Than 14 Days — Supply Chain Security Tightened

📰 Article (English)

The Python Package Index (PyPI) has begun enforcing a new policy: file uploads to any existing release that is older than 14 days are now rejected. The change was driven by supply chain security concerns — specifically, the pattern where attackers compromise a maintainer's account and silently upload a malicious file to an old, trusted version of a package that developers are still pinning. By closing this window, PyPI eliminates a common attack vector without requiring maintainers to yank or deprecate the original release. For legitimate cases such as adding a new wheel for a newly supported platform, the 14-day window still applies after release publication. Maintainers who need to add files to older releases must now contact PyPI support directly. This is part of a broader effort by the Python Software Foundation (PSF) and PyPI to harden the package ecosystem following several high-profile supply chain attacks on open source packages in recent years. Developers who ship packages are advised to review their release workflows and test cross-platform wheel uploads before the 14-day window closes.

Với bản cập nhật này, developer có thể nhấn một nút trên comment review và Copilot sẽ tạo ra một bản sửa code nhắm vào vấn đề cụ thể đó, sau đó có thể xem xét và áp dụng ngay từ giao diện mobile. Tính năng này nhắm vào một điểm ma sát phổ biến trong phát triển hiện đại: các comment review chất đống trong khi developer đang vắng mặt khỏi máy tính. GitHub engineering lưu ý rằng bản sửa được giới hạn trong vùng code cụ thể được đánh dấu trong comment, thay vì thực hiện viết lại rộng hơn. Điều này bổ sung cho tính năng "Fix with Copilot" trên web đã có sẵn tại github.com. Bản cập nhật đang được triển khai cho GitHub Mobile trên cả iOS và Android.
Hữu ích khi bạn nhận được code review nhỏ lẻ và muốn phản hồi nhanh mà không cần mở laptop. Tuy nhiên, với các thay đổi phức tạp hơn hoặc cần hiểu ngữ cảnh rộng hơn thì vẫn nên dùng IDE để kiểm soát tốt hơn.

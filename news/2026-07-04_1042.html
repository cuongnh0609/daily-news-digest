B1Generative UIとOpenUI――AIがレイアウトまで生成する新パラダイム、トークン67%削減の秘訣

📰 本文 + 和越対訳

「Generative UI」とは、AIがテキストを出力するだけでなく、UIのレイアウトや見た目そのものを動的に生成するアプローチだ。Zennで公開されたエンジニア解説記事では、その実現フレームワークである「OpenUI」の仕組みを体系的に紹介している。従来のマークダウンやJSONによるUI記述と違い、OpenUIは「OpenUI Lang」という独自言語でUIを表現し、JSON比で最大67.1%少ないトークンで同等のUIを生成できるという。生成方式には「Static（静的）」「Declarative（宣言型）」「Open-Ended（自由生成）」の3種類があり、実際の開発では制約度合いに応じて使い分けることが推奨されている。実装例では、ZodによるTypeScript型定義→LLMがOpenUI Langを出力→パーサー→Renderer描画、というパイプラインが示されており、AIエージェントがフロントエンドの振る舞いを自律的に構成できる将来像が描かれている。

"Generative UI" là cách tiếp cận mà AI không chỉ xuất ra văn bản mà còn tự động sinh cả layout và giao diện người dùng. Bài viết trên Zenn giới thiệu framework "OpenUI" để hiện thực hóa điều này. Khác với mô tả UI bằng Markdown hay JSON thông thường, OpenUI dùng ngôn ngữ riêng "OpenUI Lang", có thể biểu đạt UI tương đương với lượng token ít hơn tới 67.1% so với JSON. Có 3 phương thức sinh UI: Static (tĩnh), Declarative (khai báo) và Open-Ended (tự do). Trong ví dụ triển khai, pipeline bao gồm: định nghĩa kiểu TypeScript bằng Zod → LLM xuất OpenUI Lang → parser → Renderer, vẽ ra bức tranh tương lai nơi AI agent tự cấu trúc giao diện frontend một cách tự chủ.

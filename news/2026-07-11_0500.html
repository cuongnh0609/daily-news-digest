B2テスト全部グリーンなのに本番で落ちた——AIエージェントコードの2つの盲点

📰 本文 + 和越対訳

AIエージェントに生成させたコードがテストスイートを全通過したにもかかわらず、本番環境でクラッシュするという問題をBoutiqueHack氏がSQLite移行プロジェクトで経験し、Zennに詳細なポストモーテムを公開した。失敗の原因は大きく2類型に分けられる。1つ目は「並行性問題」で、GETエンドポイントが内部でwriteトランザクションを発行していたため、シングルスレッドのテストでは検出できなかったが、本番では複数プロセスが同時に動作してデッドロックが発生した。2つ目は「データ量問題」で、テスト用フィクスチャが小規模すぎたため、フルスキャンがテスト環境では問題なかったのに本番では深刻なボトルネックになった。著者は「AIエージェントはテストが通る状態を最適化するが、本番相当の並行数・データ量を要件として明示しない限り、それらは考慮されない」と指摘し、テスト設計の前提条件の見直しを強く促している。

BoutiqueHack đã chia sẻ post-mortem về dự án migrate SQLite sử dụng AI agent: toàn bộ test pass nhưng production lại crash. Tác giả kết luận: AI agent tối ưu hóa để test pass, nhưng nếu không nêu rõ yêu cầu về concurrency và data volume tương đương production trong spec, agent sẽ không xét đến chúng.

C3AI Cannot Forget and Does Not Know How to Forgive — Why Machine Memory Architecture Prevents Graceful Decay

📰 Article (English)

A deeply technical philosophical essay by Tejas Sudarshan argues that AI systems lack two related but distinct capabilities that humans take for granted: the ability to forget and the ability to forgive. These aren't emotional shortcomings — they're structural properties of how current AI architectures store and retrieve information. The author identifies four memory types in AI systems: weights (training parameters — nearly permanent), context window (binary, zero or full fidelity), retrieval store (embeddings at constant fidelity, never emotionally charged), and logs (distributed, fully backed up). Critically, none of these have graceful decay. Human memory uses reconsolidation — every time you recall something, the memory becomes briefly editable and can be emotionally recontextualized. Machine memory is read-only: a vector returns unchanged every time it's retrieved. Machine unlearning — the attempt to remove training data from model weights — is described as "unsolved at any real scale." And even if it were solved, forgetting alone isn't forgiveness. The essay defines true forgiveness as: "keep the record at full fidelity, and stop letting it drive behavior." No current architecture implements this semantic layer. The root cause is economic: biological memory has metabolic cost, creating evolutionary pressure to prune. Compute storage is near-free, so there's no engineering incentive to build selective forgetting. The essay closes by suggesting this could be a fundamental capability gap as AI systems take on more interpersonal roles.

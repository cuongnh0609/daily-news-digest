📰 本文 + 和越対訳

Hơn 110 học sinh trung học và giáo viên đang tham gia chuyến đi thực tế đã phát sinh các triệu chứng ngộ độc thực phẩm tại khách sạn lưu trú. B-01VercelのOSSエージェントフレームワーク「Eve」——ファイル配置だけでツール・サブエージェントが自動配線

📰 本文 + 和越対訳

Vercelが2026年6月に公開したオープンソースのエージェントフレームワーク「Eve」は、「ファイルシステム・ファースト設計」を核心コンセプトに掲げる。agent/ディレクトリ配下のtools/・skills/・subagents/といったサブフォルダにファイルを置くだけで、各構成要素が自動的に発見・配線される仕組みだ。最小構成はagent.tsとinstructions.mdの2ファイルのみで済み、開発者はビジネスロジックに集中できる。「持続的実行」「サンドボックス」「人間による承認ゲート」「評価機能」がビルトインされており、本番運用に不可欠なボイラープレートコードを大幅に削減できる点が高く評価されている。実際に記事を自動生成するエージェントを構築した著者は、Next.jsがWebフレームワーク開発にもたらした恩恵をエージェント開発に拡張したものだと評している。

Vercel đã công bố framework agent mã nguồn mở "Eve" vào tháng 6 năm 2026, với khái niệm cốt lõi là "thiết kế ưu tiên filesystem". Chỉ cần đặt file vào các thư mục con như tools/, skills/, subagents/ bên trong thư mục agent/, các thành phần sẽ tự động được phát hiện và kết nối với nhau. Cấu hình tối thiểu chỉ cần 2 file là agent.ts và instructions.md, giúp developer tập trung vào business logic. Framework tích hợp sẵn các tính năng như "persistent execution", "sandbox", "human approval gate" và "evaluation", giúp giảm đáng kể boilerplate code cho môi trường production. Tác giả bài viết đánh giá đây là sự mở rộng những gì Next.js làm cho web framework sang lĩnh vực phát triển agent.

C-03Project Valhalla Lands in JDK 28: A Decade of Work Brings Value Types to Java

📰 Article (English)

After nearly ten years of development, Project Valhalla — one of the most ambitious JVM evolution efforts ever undertaken — is arriving in JDK 28. The project fundamentally changes Java's memory model by introducing "value types": user-defined primitives that behave like int or double — no object identity, no null, no heap allocation by default. The core innovation is the elimination of the "object tax" in Java. Currently, wrapping a primitive like int in a class (e.g., Integer) forces heap allocation and pointer indirection, which hurts performance in tight loops and data-intensive workloads. Value types allow complex types like Point, Complex, or Money to be stored inline in arrays and on the stack — dramatically improving cache locality. For backend engineers, this means numeric-heavy code (financial calculations, vector math, ML inference preprocessing) can achieve C++-level performance without leaving the JVM. The JIT compiler can also eliminate boxing/unboxing overhead across generics. JDK 28 is expected as a preview feature, with stabilization in JDK 29–30. Cloud engineers running Java microservices on Kubernetes will see reduced heap pressure and lower GC pause times — measurable improvements in p99 latency for services processing large data payloads.

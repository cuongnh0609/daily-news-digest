B1自作MCPサーバーのトークン消費を9割削減する「退避パターン」— 大量データをコンテキスト外に逃がす設計手法

📰 本文 + 和越対訳

Model Context Protocol（MCP）を利用したAIエージェントの実装において、避けられない課題がある。それは、MCPサーバーがファイル内容やDBクエリ結果といった大量データをそのままAIのコンテキストに流し込んでしまうことによる、トークン消費の爆増だ。Air ClosetのCTO・辻亮佑氏が提唱するのが「退避パターン」と呼ばれる設計手法で、大容量データをMCPの通信チャネル外に移し、URLやキーなどの参照情報のみをAIに渡すというアプローチだ。例えばソースコードの変更を扱う場合、コード全体をコンテキストに含める代わりにGitリポジトリのURLを渡して`git push`させる方法や、DBクエリ結果が数千行に及ぶ場合は自動的にGoogle SheetsにエクスポートしてそのシートURLのみを返す方法がある。この手法により、ツール全体のトークン消費を70〜90%削減できると報告されている。Google WorkspaceのOAuth統合を活用することでセキュリティも担保でき、実用的なコスト最適化策として注目を集めている。

Trong quá trình triển khai AI agent sử dụng Model Context Protocol (MCP), có một thách thức không thể tránh khỏi: việc MCP server đưa trực tiếp lượng dữ liệu lớn như nội dung file hay kết quả query DB vào context của AI khiến lượng token tiêu thụ bùng nổ. CTO của Air Closet, ông Tsuji Ryosuke đề xuất phương pháp thiết kế gọi là "evacuation pattern" (退避パターン): di chuyển dữ liệu lớn ra ngoài kênh truyền thông của MCP và chỉ truyền thông tin tham chiếu như URL hay key cho AI. Ví dụ khi xử lý thay đổi source code, thay vì đưa toàn bộ code vào context, chỉ cần truyền URL của Git repository và để AI thực hiện git push. Khi kết quả query DB có hàng nghìn dòng, tự động export sang Google Sheets và chỉ trả về URL của sheet đó. Phương pháp này được báo cáo có thể giảm tới 70-90% lượng token tiêu thụ trên toàn bộ tool. Bằng cách tận dụng tích hợp OAuth của Google Workspace, bảo mật cũng được đảm bảo, thu hút sự chú ý như một giải pháp tối ưu hóa chi phí thực tiễn.

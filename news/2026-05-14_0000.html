B1Claude Codeに長期記憶を——自作ツール「kita-memory」の3層アーキテクチャ

📰 本文 + 和越対訳

Claude Codeはセッションをまたいで記憶を保持しない——この根本的な制約を解決しようとした開発者が、Supabaseとベクトル検索を組み合わせた自作ツール「kita-memory」を公開した。 アーキテクチャは大きく3つのコンポーネントに分かれる。Save CLIはClaude Codeの会話ログをSupabaseのmemoriesテーブルに同期書き込みし、Consolidate Edge FunctionがGemini Flashで要約したうえでベクトル化を非同期実行する。 MCP Serverは読み取り専用で、Claudeが過去の記憶を問い合わせるためだけに使われる——書き込みはCLIのみが担うことで、AIが自分の記憶を上書き・破損するリスクをゼロにしている。 検索精度の向上にはpgvectorのコサイン類似度とpg_trgmのトライグラムマッチングを「Reciprocal Rank Fusion」で融合するハイブリッド手法を採用し、単純なベクトル検索より大幅に再現率が向上した。 埋め込みの次元数は768で固定しており、将来モデルを変更しても互換性が壊れないよう設計されている点もこだわりのひとつだ。

Một nhà phát triển đã công bố công cụ tự làm "kita-memory" để giải quyết hạn chế cơ bản của Claude Code: không lưu trữ ký ức giữa các session. Kiến trúc gồm 3 thành phần chính: Save CLI ghi đồng bộ log hội thoại Claude Code vào bảng memories trên Supabase; Consolidate Edge Function tóm tắt bằng Gemini Flash rồi vector hóa bất đồng bộ; MCP Server chỉ đọc, dùng để Claude truy vấn ký ức cũ — việc phân tách đọc/ghi giúp loại bỏ hoàn toàn rủi ro AI tự ghi đè ký ức của mình. Để tăng độ chính xác tìm kiếm, hệ thống kết hợp độ tương đồng cosine của pgvector và trigram matching của pg_trgm thông qua kỹ thuật "Reciprocal Rank Fusion", cải thiện đáng kể recall rate so với vector search đơn thuần. Kích thước embedding cố định ở 768 chiều để đảm bảo tương thích ngay cả khi thay đổi model trong tương lai.

B3Claude Code内部解剖 第4章——6段階「コンテキスト圧縮パイプライン」の設計思想

📰 本文 + 和越対訳

Claude Codeのソースコードを徹底解析するシリーズ第4章として、エージェントループにおけるコンテキスト管理の全体設計が公開された。 ファイル読み込みやコマンド実行が積み重なるほどコンテキストは指数関数的に膨張するが、Claude Codeはこれを6段階のスケジューリングパイプラインで制御している。 まず「Tool Result Budgeting」で個々のツール出力に上限を設け、次に「Snipping」で古いブロックをマーカーに置換、「MicroCompact」で陳腐化したファイル読み込みなどを除去する。 さらに「Context Collapse」で構造を非破壊的に畳み込み、それでも不足なら「AutoCompact」がタスク引き継ぎメモを生成し、最後の砦として「Reactive Compact」がトークン推定失敗時の緊急回収を担う。 圧縮後も直近のメッセージ「テール」を必ず再付加することで、エージェントは会話の継続感を失わない設計になっている。

Chương 4 của loạt bài phân tích mã nguồn Claude Code đã được công bố, tiết lộ thiết kế tổng thể của hệ thống quản lý context trong vòng lặp agent. Khi các lệnh đọc file và thực thi tích lũy, context tăng trưởng theo cấp số nhân — Claude Code kiểm soát điều này qua pipeline lập lịch 6 tầng. Đầu tiên "Tool Result Budgeting" giới hạn kích thước đầu ra từng tool; tiếp theo "Snipping" thay thế các khối cũ bằng marker; "MicroCompact" loại bỏ kết quả đọc file đã lỗi thời. Tiếp theo "Context Collapse" gấp cấu trúc không phá hủy; nếu vẫn chưa đủ thì "AutoCompact" tạo memo bàn giao task; cuối cùng "Reactive Compact" xử lý khẩn cấp khi ước tính token thất bại. Sau khi nén, "đuôi" tin nhắn gần nhất luôn được gắn lại để agent không mất cảm giác liên tục của cuộc hội thoại.

C3Needle: A 26M-Parameter Tool-Calling Model Distilled from Gemini — 6,000 Tokens/sec on Edge Devices

📰 Article (English)

Cactus Compute has open-sourced Needle, a 26 million parameter model purpose-built for single-shot function calling, distilled from Gemini 3.1. The project targets a specific gap in the agentic AI stack: on-device tool dispatch for phones, watches, and resource-constrained hardware. The architecture — called Simple Attention Network (SAN) — strips out feed-forward networks entirely, using 12 encoder layers (self-attention + gated residuals) plus 8 decoder layers (masked self-attention + cross-attention), with 512 embedding dimensions, 8 attention heads, and a compact 8,192-token BPE vocabulary. Training ran on 16 TPU v6e chips: 200 billion tokens of general pretraining over 27 hours, followed by 2 billion tokens of function-calling-specific post-training in just 45 minutes. Performance numbers are striking for the model size: 6,000 tokens/second prefill and 1,200 tokens/second decode on Cactus hardware. The model outperforms FunctionGemma-270m and Qwen-0.6B on standard tool-calling benchmarks despite being roughly 10× smaller than some competitors. Weights are available on Hugging Face under an open license. The release includes a CLI, web playground, and local fine-tuning support. For agentic system designers, Needle represents a viable path to routing low-complexity tool calls to an on-device model, reserving cloud inference for complex reasoning.

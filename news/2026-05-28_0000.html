Claude Code 向けスキルとして公開された「cc-rsg（Reverse Spec Generator）」は、「仕様書からコードを生成する」という従来の AI 活用を逆転させ、「既存コードから仕様書を自動生成する」アプローチを実現するツールだ。動作は 6 フェーズで構成されており、まず 5 つの対話的な質問（読み手の属性・用途・粒度・注意領域・出力テンプレート）で仕様書の方向性を決定する。その後、コードベースの偵察、インベントリ（抽出単位）の作成、並列サブエージェントによるドラフト生成、カバレッジ検証、未解決事項の対話、そして最終成果物の納品という流れで進む。生成された仕様書には「[REF: file.js:142]」という形でソースコードへの参照が付与され、信頼度マーカー（HIGH/MED/LOW）と仮定の明示により記述の確実性を区別している。実測では、テトリス（267 行）で約 15 分で 10 章のドラフトが完成。大規模コード（Redmine、10 万行）でも「コードベース全体の地図」を生成できる実力を見せた。

"cc-rsg (Reverse Spec Generator)" — một skill cho Claude Code — đã đảo ngược cách tiếp cận AI thông thường: thay vì "tạo code từ spec", nó "tự động tạo spec từ code hiện có". Cloudflare đã ra mắt Flagship — một dịch vụ quản lý feature flag được quản lý hoàn toàn, giúp developer kiểm soát chi tiết hành vi ứng dụng mà không cần deploy lại code. Dịch vụ này tích hợp trực tiếp vào Cloudflare Workers thông qua cơ chế binding, cung cấp đánh giá flag an toàn kiểu dữ liệu với tự động fallback về giá trị mặc định. Flagship hỗ trợ 11 toán tử targeting khác nhau, cho phép phục vụ flag khác nhau dựa trên thuộc tính người dùng với các quy tắc kết hợp qua AND/OR logic. Cho các rollout dần dần, dịch vụ sử dụng consistent hashing để đảm bảo mỗi người dùng luôn nhận cùng giá trị flag qua các session — quan trọng cho A/B testing và canary release. Flag hỗ trợ boolean, string, number và JSON object, cho phép gửi toàn bộ khối cấu hình qua một flag duy nhất. Điểm khác biệt chính là tuân thủ tiêu chuẩn mở OpenFeature của CNCF, làm cho Flagship tương thích với SDK @cloudflare/flagship trên Workers, Node.js và browser. Thiết kế provider-agnostic này có nghĩa là team có thể thay đổi provider feature flag mà không cần sửa code evaluation. Flag được quản lý qua Cloudflare dashboard, tổ chức theo app cho từng project. Flagship hiện đã generally available cho người dùng Cloudflare Workers.
Nếu bạn đang dùng Cloudflare Workers, Flagship giúp loại bỏ dependency vào dịch vụ feature flag bên ngoài (LaunchDarkly, Split.io). Tuân thủ OpenFeature nghĩa là code evaluation có thể tái sử dụng dù sau này bạn đổi sang provider khác.

Consistent hashing /kənˈsɪstənt ˈhæʃɪŋ/ Hashing nhất quán — thuật toán đảm bảo cùng một input (user ID) luôn map đến cùng một output (flag value), dù server pool thay đổi. Quan trọng cho A/B test ổn định.
OpenFeature /ˈəʊpən ˈfiːtʃər/ Tiêu chuẩn mở của CNCF cho feature flag — định nghĩa interface chung để evaluation code hoạt động với bất kỳ provider nào, tránh vendor lock-in.
Canary release /kəˈnɛəri rɪˈliːs/ Release từng bước — deploy tính năng mới cho một phần nhỏ người dùng trước, quan sát tác động, rồi mới mở rộng dần. Giảm rủi ro so với deploy toàn bộ.

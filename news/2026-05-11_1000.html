B1Spec KitでAI駆動開発を始める——Claude CodeとGitHub Copilot両対応の仕様ファースト開発ガイド

📰 本文 + 和越対訳

GitHubが公開したSpec Kitは、AIエージェントに対して「仕様書を書いてからコードを書く」という原則を徹底させるためのフレームワークだ。従来のAI駆動開発では、曖昧な指示をそのままプロンプトに渡すことでコードの品質が不均一になるという課題があった。Spec Kitでは`spec/`ディレクトリに機能仕様・データモデル・APIコントラクトを記述し、Claude CodeやGitHub Copilotがそのspecファイルを参照しながら実装を進める設計になっている。Zennの解説記事によると、specに変更が生じた場合はエージェントが差分を検出して影響範囲を提示し、人間は意思決定だけに集中できるフローが実現されるという。このアプローチはSpec-Driven Developmentと呼ばれ、AIエージェントの暴走を防ぐ「文脈の固定化」として注目を集めている。

Spec Kit do GitHub phát hành là framework giúp các AI agent tuân thủ nguyên tắc "viết spec trước, code sau". Trong AI-driven development truyền thống, việc truyền các hướng dẫn mơ hồ trực tiếp vào prompt thường dẫn đến chất lượng code không đồng đều. Với Spec Kit, các spec chức năng, data model và API contract được viết trong thư mục `spec/`, và Claude Code cùng GitHub Copilot sẽ tham chiếu các file spec này trong quá trình implement. Khi spec thay đổi, agent phát hiện diff và liệt kê phạm vi ảnh hưởng để con người chỉ cần tập trung vào quyết định. Cách tiếp cận này gọi là Spec-Driven Development, đang được chú ý như phương pháp "cố định ngữ cảnh" để kiểm soát AI agent.

B2gbrainに学ぶAgentの記憶基盤——Knowledge GraphとMCPで実現する多層メモリアーキテクチャ

📰 本文 + 和越対訳

Y Combinator代表が公開したオープンソースプロジェクト「gbrain」は、AIエージェントに永続的な記憶基盤を与えるシステムだ。データはPostgreSQLまたはPGLiteに保存され、MCPツール経由でエージェントが知識にアクセスし、グラフを巡回できる設計になっている。コアデータモデルは個々の知識エントリを表す`pages`テーブルと検索単位となる`content_chunks`で構成され、さらにエンティティ間の型付き関係を記録する`links`テーブルが関係を管理する。ナレッジグラフはLLMによる抽出に頼らず、Markdownリンク・WikiリンクやYAMLフロントマターのフィールドを正規表現で解析して構築するため、決定論的かつ改ざん耐性が高い。グラフ検索にはSQLの再帰CTEを使い、サイクル防止を施した多段トラバーサルを実現している。

"gbrain" là dự án open-source do CEO của Y Combinator công bố, cung cấp cơ sở hạ tầng bộ nhớ bền vững cho AI agent. Dữ liệu được lưu trong PostgreSQL hoặc PGLite, và agent truy cập kiến thức qua MCP tools để duyệt qua knowledge graph. Data model cốt lõi gồm bảng `pages` (entry kiến thức) và `content_chunks` (đơn vị tìm kiếm), kết hợp bảng `links` quản lý quan hệ có kiểu dữ liệu giữa các entity. Knowledge graph được xây dựng bằng regex phân tích Markdown links, wikilinks và YAML frontmatter — không dùng LLM để trích xuất — đảm bảo tính xác định và chống giả mạo. Tìm kiếm đồ thị sử dụng recursive CTE của SQL với cơ chế phòng chu kỳ để thực hiện multi-hop traversal.

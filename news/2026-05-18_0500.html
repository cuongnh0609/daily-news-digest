C1Anthropic Engineering: Scaling Managed Agents — Decoupling the Brain from the Hands

📰 Article (English)

Anthropic's engineering blog published a deep-dive into their architecture for scaling managed agents, specifically addressing how to decouple the "brain" (the LLM performing reasoning and planning) from the "hands" (tool-execution workers that carry out actions). The central insight is that long-running agentic tasks encounter a fundamental bottleneck: the reasoning model needs to stay in the loop for decisions, but blocking it during tool execution wastes expensive compute. The proposed architecture introduces asynchronous execution workers that independently carry out tool calls — file reads, API requests, bash commands — and return results to a lightweight orchestrator. The orchestrator batches results and presents them to the LLM only when a decision is needed, dramatically increasing throughput. The post details practical tradeoffs around state management, error recovery, and the granularity at which tool batches are presented to the model. Key finding: the optimal "decision batch size" depends heavily on task type — coding agents benefit from larger batches while customer-service agents require tighter loops. This architecture is now used in Claude Code's background agent mode.

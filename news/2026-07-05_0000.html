Next.jsを用いた本番環境で「Cannot convert argument to a ByteString」というエラーが断続的に発生し、原因の特定に苦労した開発者の調査記録だ。 問題の根本原因は、publicディレクトリに配置されていた「チキン.png」という日本語ファイル名にあった。 本番環境ではNext.jsがクライアントコンポーネントの事前読み込みヒントをHTTPのLinkレスポンスヘッダーに埋め込む仕様があり、HTTPヘッダーはByteString（文字コード0〜255の範囲）しか受け付けないため、日本語文字がこの制限を超えてNode.jsのundiciライブラリがエラーを出す。 開発環境ではpreloadヒントがHTMLの<link>タグに出力されるためUTF-8が問題なく通るが、本番環境はHTTPヘッダーを使うため同じファイルでも動作が異なる。 React Routerはpreloadを常にHTMLに出力するため同問題は発生しないという点も解説されており、解決策は至ってシンプル——ファイル名を英数字とハイフンのみに変更することだ。

Ở môi trường production, Next.js nhúng các preload hint cho client component vào HTTP response header Link — nhưng HTTP header chỉ chấp nhận ByteString (ký tự 0–255), khiến ký tự tiếng Nhật vượt giới hạn này và thư viện undici của Node.js báo lỗi. Môi trường development xuất preload hint ra thẻ HTML <link> nên UTF-8 không vấn đề, còn production dùng HTTP header nên hành vi khác nhau. React Router không gặp vấn đề này vì luôn xuất preload vào HTML. Giải pháp đơn giản: đổi tên file chỉ dùng chữ cái, số và dấu gạch ngang.

Cursor's engineering team published details of an innovative training technique they call "autoinstall" — a bootstrapping method where an earlier model version (Composer 1.5) is used to prepare working environments for training the next generation model (Composer 2). The core problem: reinforcement learning training requires runnable environments, and broken setups waste compute without producing useful training signals. The autoinstall system runs as a two-stage pipeline. First, an agent proposes ten commands and their expected outputs for correct environment setup, examining READMEs, Makefiles, and package manager configs. Second, a separate agent attempts to execute three selected commands, iterating up to five times before discarding broken setups. The system handles missing dependencies creatively — mocking files, creating placeholder images, even fabricating database tables. For the blockchain project Celo, the agent discovered undocumented installation commands via web search and mocked an authentication flow to get the test suite running. Results: Composer 2 scored 61.7% on Terminal-Bench (environment setup benchmark), up from Composer 1.5's 47.9% — a 13.8-point improvement directly attributable to this training data quality. The team sees future applications in run management, data preprocessing, and architecture tuning.

C3Tilde.run: Transactional Versioned Filesystem for AI Agent Sandboxes — Built by Team Behind lakeFS

📰 Article (English)

A new platform called Tilde.run has emerged from stealth, built by the team behind lakeFS (the open-source data versioning tool). The core proposition: every AI agent run is a fully reversible transaction on a versioned, composable filesystem — analogous to git for agents. Data sources (GitHub repositories, S3 buckets, Google Drive) are mounted as a unified ~/sandbox directory. When an agent run succeeds, changes are committed; when it fails, the entire filesystem state is discarded atomically. Each run produces a full timeline with diffs, commit history, and instant rollback capability. The security model is default-deny network policy — all outbound requests from agents are policy-checked and logged. RBAC provides granular per-agent permission policies separate from user credentials, meaning a compromised agent cannot escalate beyond its defined scope. Integration targets include Claude (via SDK), AWS S3, Hugging Face model hub, LangGraph workflows, and Google Drive. Installation is a single curl command. The platform is available via CLI and Python SDK. This addresses a real gap: as AI agents handle longer-horizon tasks touching production data, teams need audit trails, rollback guarantees, and permission isolation that traditional file systems don't provide.

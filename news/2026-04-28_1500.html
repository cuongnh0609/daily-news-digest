B-1非エンジニアの「作りたい」と「安全に公開したい」を両立する Sandbox MCP を作った

📰 本文 + 和越対訳

エアークローゼットCTOの辻亮介氏が、非エンジニアでも Claude Code を使って社内アプリを安全に公開できる統合プラットフォーム「Sandbox MCP」を開発・公開した。従来、PM やデザイナーなどの非技術系スタッフが AI ツールでアプリを作れるようになっても、インフラ設定や認証・データ管理などのセキュリティ課題が deploy の壁になっていた。Sandbox MCP では、アプリを単一コマンドで `https://sbx-{nickname}--{app-name}.example.com/` に deploy できる仕組みを提供し、セキュリティを4層構造で担保している。具体的には、Cloudflare ZeroTrust OAuthによる認証、MCP OAuth によるオペレーション管理、Firestoreによる名前空間分離、および制限的な Cloud Run IAM 権限の組み合わせにより、想定外のアクセスを防ぐ。UIは `sandbox-ui-kit` で標準化され、バックエンドは Python/Node.js/静的HTML/カスタム Docker を自動判別して対応する。ノンエンジニアが安心してAIツールを活用できる環境整備として、業界から高い関心が寄せられている。

CTO của Air Closet, Tsuji Ryosuke, đã phát triển và công bố "Sandbox MCP" — một nền tảng tích hợp giúp người không phải kỹ sư có thể tạo và deploy ứng dụng nội bộ một cách an toàn bằng Claude Code. Trước đây, dù các nhân viên phi kỹ thuật như PM hay designer có thể tạo app với AI, họ vẫn gặp trở ngại lớn khi deploy do các vấn đề về cấu hình infrastructure, xác thực và bảo mật dữ liệu. C-2pip 26.1 Ships Native Lockfiles and Dependency Cooldowns — Python Package Management Transformed

📰 Article (English)

pip 26.1 has landed with two major features that fundamentally change Python dependency management: native lockfile support and dependency cooldowns. The new pip lock command generates a pylock.toml file capturing the complete, resolved dependency tree — similar to what npm ci and Cargo have provided for years, but now natively in pip without third-party tools like pip-tools or Poetry. The second feature, --uploaded-prior-to, introduces a supply chain security mechanism called "dependency cooldowns." Using ISO 8601 duration format, developers can restrict installations to packages uploaded before a specified period. For example, pip install --uploaded-prior-to P4D llm installs only versions at least 4 days old, preventing immediate adoption of potentially compromised newly-released packages. This addresses a real threat vector where attackers publish malicious packages hoping for fast adoption before detection. The release also drops Python 3.9 support (EOL October 2025). Simon Willison tested the features with Datasette and the LLM CLI tool, confirming the generated pylock.toml correctly captured 519 lines of resolved dependencies.

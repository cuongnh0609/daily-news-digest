C2GitHub Secret Scanning Now Monitors All Public GitHub Content for Leaked Enterprise Credentials

📰 Article (English)

GitHub has launched "public monitoring" for enterprises with Secret Protection, a new feature that scans the entirety of github.com's public surfaces — git content, pull request comments, and issues — for leaked secrets belonging to enterprise organizations. Previously, secret scanning primarily protected private repositories; this extends coverage to public exposure pathways that developers commonly overlook: personal forks of private repos, open-source contributions using company credentials, and discussions in public issues. The feature attributes findings back to the enterprise using two methods: member-based attribution (matching the GitHub account of a known enterprise member) and verified domain matching (catching company-email users who lack formal enterprise affiliation). Enabling public monitoring requires no additional configuration — enterprise owners activate it from the Security tab and the system immediately surfaces recently leaked secrets and begins monitoring for future matches. Validity checking is also built in, determining whether a discovered secret is still active. Critically, the feature operates at no extra cost for enterprises already subscribed to Secret Protection. This is a meaningful shift: secrets have no respect for private/public boundaries, and attackers actively scan public GitHub content for leaked credentials within minutes of exposure.

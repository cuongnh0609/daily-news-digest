Google has published a technical deep-dive on accelerating Gemma 4 inference using multi-token prediction (MTP) draft models, achieving substantial latency reductions while maintaining output quality. The technique pairs a large target model with a smaller, faster draft model that speculatively generates multiple tokens ahead; the target model then verifies and accepts correct tokens in a single forward pass, reducing the total number of sequential generation steps required. In benchmark evaluations, the MTP drafter approach delivers up to a 3× reduction in time-to-first-token on consumer-grade and cloud inference hardware. The draft models are purpose-trained as companion artifacts to Gemma 4 and are released alongside the main model weights, meaning developers do not need to find or train their own speculative decoding partners. The release includes detailed guidance on integrating MTP drafters with popular inference frameworks including vLLM, Ollama, and Hugging Face's text-generation-inference (TGI). Google notes that the approach is particularly effective for long-context tasks such as document summarization and code generation, where sequential token generation is the primary bottleneck. This development is significant for teams running Gemma 4 locally or on cost-sensitive cloud deployments, as it directly addresses the inference cost barrier that has limited widespread adoption of capable open models.

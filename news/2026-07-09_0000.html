📰 Article (English)

The Model Context Protocol (MCP), now under the Linux Foundation, has promoted its Enterprise-Managed Authorization (EMA) extension to stable status. This is a significant milestone for organizations deploying MCP at scale, as it solves one of the biggest practical friction points: per-user, per-server manual authentication. Previously, every employee needing to use an MCP-enabled tool had to authenticate individually with each server — a major obstacle for enterprise adoption. EMA changes the architecture fundamentally. Users authenticate once through their organization's existing Identity Provider (IdP), such as Okta, and the system automatically provisions access to all MCP servers their role permits. Access policies are managed centrally, audit logs are unified, and the security team gains visibility across the entire tool surface. Currently supported IdPs include Okta, while compatible clients include Claude and Visual Studio Code. Server-side support extends to enterprise tools like Asana, Atlassian, Canva, Figma, and Linear, with Slack integration announced as coming soon. Per-user budgets for cost centers are also now available through an accompanying GitHub Copilot billing update released the same week. For teams already using MCP for internal tooling, this stabilization removes the last major blocker for rolling out AI agents to broader employee bases without requiring per-person credential management.

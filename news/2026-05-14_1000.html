📰 本文 + 和越対訳

NOT A HOTELのエンジニアuji氏が、Claude Code Routinesを活用してGitHub Issueを自動対応するチーム運用の詳細を公開した。4人のエンジニアがそれぞれ自分のアカウントで/auto-fix-issueスキルを設定し、平日の毎時実行（実行時刻を15分ずつずらして実質15分間隔）というスケジュールで運用している。Routinesが起動すると、まずgh issue listでオープンのIssueを取得し、すでにPRが存在するものをスキップした後、対応可能な1件を選んでブランチを作成、最小限の変更でコミット・プッシュ・PR作成まで自律的に完遂する。CI失敗時は追加コミットで自動修正するループも動作する。Routinesは「複雑な判断・コード変更」を担当し、GitHub Actionsは「ラベリング・アサイン・デプロイなどの決定論的タスク」を担当するという明確な役割分担が設計のポイントだ。チームはスマートフォンからIssueを投げるだけでPRが自動生成される体験を「心理的ハードルの劇的な低下」と評価している。

Kỹ sư uji tại NOT A HOTEL chia sẻ chi tiết về việc vận hành tự động xử lý GitHub Issue theo nhóm bằng Claude Code Routines. 4 kỹ sư mỗi người thiết lập skill /auto-fix-issue trên tài khoản của mình, chạy mỗi giờ vào ngày thường với thời gian lệch nhau 15 phút, tạo ra hiệu quả xử lý thực tế mỗi 15 phút. Khi Routine khởi động, nó lấy danh sách Issue mở, bỏ qua các Issue đã có PR, chọn một Issue có thể xử lý để tạo branch, commit, push và tạo PR một cách tự động. Vòng lặp tự động sửa lỗi CI cũng hoạt động khi build thất bại. Điểm thiết kế then chốt là phân chia rõ vai trò: Routines đảm nhận "phán đoán phức tạp và thay đổi code", còn GitHub Actions xử lý "các tác vụ xác định luận như gán nhãn, assign và deploy". Nhóm đánh giá trải nghiệm tạo PR tự động chỉ bằng cách đăng Issue từ điện thoại là "giảm đáng kể rào cản tâm lý".

Framework AI agent mã nguồn mở Mastra chính thức thông báo hỗ trợ Temporal Workflows (ngày 12/5/2026).

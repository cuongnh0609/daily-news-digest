B1LangSmith Engineがエージェント開発を自動化——本番トレースからバグ・修正PRを自動生成

📰 本文 + 和越対訳

LangChainは2026年5月13日、エージェントの開発サイクルを自動化するツール「LangSmith Engine」をパブリックベータとして公開した。このツールは本番環境のトレースデータを継続的に監視し、失敗パターンを自動で検出・クラスタリングして根本原因を分析する。従来、開発者がトレースを手動で読み込んでバグを特定するという作業に数時間を費やしていたプロセスを、Engineが自動化する。具体的には、エラー・オンライン評価器の失敗・異常なトレース・ユーザーフィードバックといった複数のシグナルを統合し、問題パターンごとに①修正コードのPull Request②カスタム評価器の作成③失敗トレースのデータセット追加という3種類のアクションを提案する。AIエージェントのプロダクション運用が増える中、人間のエンジニアの代わりにエージェント自身が自らの問題を継続的に改善していく「自己改善ループ」の実現に向けた重要な一歩といえる。

LangChain đã ra mắt "LangSmith Engine" dưới dạng public beta vào ngày 13 tháng 5 năm 2026 — một công cụ tự động hóa vòng lặp phát triển agent. Engine liên tục theo dõi dữ liệu trace từ môi trường production, tự động phát hiện và phân nhóm các pattern lỗi, rồi phân tích nguyên nhân gốc rễ. Thay vì kỹ sư phải dành hàng giờ đọc thủ công các trace để tìm bug, Engine tổng hợp nhiều loại tín hiệu (lỗi, đánh giá thất bại, phản hồi người dùng) và đề xuất 3 hành động: tạo Pull Request sửa code, tạo evaluator tùy chỉnh, và bổ sung trace lỗi vào dataset đánh giá. Đây là bước tiến quan trọng hướng tới "self-improvement loop" — nơi agent tự cải thiện mình liên tục thay vì phụ thuộc hoàn toàn vào con người.

C1KanBots: Open-Source Kanban Board That Dispatches Parallel Claude/Codex Agents per Card

📰 Article (English)

KanBots is a desktop-first, open-source project management tool that treats each Kanban card as a task dispatched to an autonomous coding agent. Instead of running a single AI session, KanBots spawns Claude Code or OpenAI Codex in isolated git worktrees — one per card — so multiple agents work in parallel without clobbering each other's state. C3CISA Data Leak: Contractor Exposed AWS GovCloud Keys and RSA Private Keys on Public GitHub

📰 Article (English)

A CISA (Cybersecurity and Infrastructure Security Agency) contractor created a public GitHub repository named "Private-CISA" in November 2025 and deliberately disabled GitHub's built-in secret scanning protections before pushing plaintext credentials — including AWS GovCloud access keys, RSA private keys, and other agency secrets — for dozens of internal systems. The breach was reported to CISA and, as of May 22, the agency was still in the process of invalidating credentials more than a week after notification. Security researcher Dylan Ayrey analyzed the exposure and concluded that an attacker with the leaked RSA private key could have read source code from every repository in the CISA-IT GitHub organization, including private ones, and registered rogue self-hosted runners to hijack CI/CD pipelines across the agency's infrastructure. Senators Maggie Hassan and Representative Bennie Thompson demanded answers, with Thompson noting that the incident reflects "diminished security culture and/or inability to adequately manage contract support." Beyond the specific breach, the incident highlights a systemic risk: cybercriminals and nation-state actors actively monitor GitHub's public commit feed for newly pushed secrets. GitHub's secret scanning would have caught this automatically — the contractor actively bypassed it. For teams managing CI/CD pipelines with government or enterprise contractors, the lesson is that access controls for third-party contributors must be hardened at the org level, not left to individual responsibility.

C2GhostLock: A 15-Year Linux Kernel Use-After-Free That Allows Privilege Escalation and Container Escapes

📰 Article (English)

Security researchers at Nebusec have published details of GhostLock (CVE-2026-43499), a critical use-after-free vulnerability in the Linux kernel's real-time mutex implementation that existed across all major distributions from kernel version 2.6.39-rc1 (circa 2011) through 7.1-rc1, patched in April 2026. The bug lives in the remove_waiter() function, which incorrectly clears the pi_blocked_on pointer on the wrong task during Requeue-PI operations. When proxy-locking fails with a deadlock condition, the cleanup logic operates on the requeuer's task instead of the actual waiter's task, leaving a dangling pointer to freed kernel stack memory. An unprivileged local attacker can exploit this to obtain that dangling pointer, write pointers to arbitrary addresses, and hijack function tables — ultimately achieving full privilege escalation or container escape without any special permissions or configuration. The vulnerability requires no complex setup: any user account is sufficient. Mitigation options include patching to a kernel version released after April 2026, or enabling RANDOMIZE_KSTACK_OFFSET in the kernel configuration to reduce exploitability. Production servers and container orchestration environments should treat this as urgent if unpatched.

🔴 HIGH Critical CVE: any unprivileged user can escalate to root or escape containers on unpatched Linux kernels (<7.1). Patch immediately.

Cực kỳ nghiêm trọng với môi trường production Linux và Kubernetes: người dùng không có đặc quyền có thể thoát container và leo thang lên root. Kiểm tra ngay phiên bản kernel của tất cả server và node đang chạy (uname -r), và ưu tiên cập nhật nếu còn dưới kernel 7.1 (patch tháng 4/2026).

Dashboard được tổ chức quanh ba trụ cột: Inbox hiển thị các mục cần xử lý (yêu cầu review, thay đổi cần thực hiện, PR sẵn sàng merge); Saved Views cho phép lưu các search query tùy chỉnh để dùng lại; và lớp tìm kiếm và lọc mạnh mẽ hỗ trợ toán tử AND/OR và query lồng nhau. Kể từ giai đoạn preview, GitHub đã bổ sung: điều hướng bàn phím theo kiểu vim bằng phím j và k, xử lý trạng thái CI và merge được cải thiện, hai filter token mới (team-review-requested-user và review-involves), và filter tác giả giờ đây có thể lọc cả PR do AI (GitHub Copilot) tạo ra. Tính năng Saved Views đặc biệt hữu ích với các team quản lý nhiều luồng review — thay vì đánh dấu trang các URL tìm kiếm phức tạp, developer có thể lưu, đặt tên và tổ chức các preset truy vấn ngay trong giao diện.
Cải thiện đáng kể cho workflow review PR hàng ngày: thêm phím tắt j/k để điều hướng PR list, và tạo Saved Views cho các filter thường dùng (VD: "PR của team tôi cần review", "PR Copilot tạo ra cần duyệt"). Đặc biệt hữu ích nếu bạn làm việc trên nhiều repo song song.

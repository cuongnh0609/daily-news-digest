📰 Article (English)

Ploy, an AI-powered marketing website builder, documented their migration from Claude Opus to GPT-5.6 Sol after four months on Claude. Top-line results: builds completed 2.2× faster (3m 42s vs. 8m), costs dropped 27%, and visual quality scores improved from 0.936 to 0.970. However, the deeper lesson is that seemingly API-compatible models hide fundamentally different behaviors that silently break production agents. Three critical incompatibilities emerged: First, GPT-5.6 fills all 25 parameters in tool calls (inventing values for unused ones like "offset: 0"), while Claude omits absent optional parameters — this caused 52–64% of file reads to return empty results until schemas were restructured with anyOf: [T, null] for nullable fields. Second, prompt caching architectures diverge: Claude caches across organizations; GPT-5.6 uses per-workspace keys with 15-request-per-minute throttling per node, and misconfiguring this initially made GPT-5.6 appear 50% more expensive. Correct layered caching achieved 83.7% hit rates. Third, server-side reasoning references failed intermittently mid-conversation until store: false was set to use self-contained encrypted content. The core engineering takeaway: "Your harness is tuned to your incumbent model, and you don't know it." Provider migration is not a configuration change — it's a full harness audit.

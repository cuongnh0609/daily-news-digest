📰 本文 + 和越対訳

📰 Article (English)

Perplexity's research team has published a paper proposing a fundamental reframing of information retrieval: rather than treating search as a ranking problem (find and order relevant documents), they argue it should be treated as a code generation problem. The core insight is that a query's "answer" is better expressed as an executable program that retrieves, aggregates, and transforms structured data from multiple sources — similar to how SQL generates results from databases rather than keyword-matching against documents. The proposed system uses an LLM to synthesize queries into "retrieval programs" — short code snippets that specify exactly which data sources to hit, how to join information across them, and what transformation to apply. These programs are deterministic and auditable: you can inspect exactly what was retrieved and why, addressing a key criticism of black-box RAG pipelines. Early benchmarks on open-domain QA tasks show significant improvements over standard RAG baselines in both accuracy and citation quality. The approach is particularly compelling for structured-data-heavy queries (e.g., "compare cloud pricing across AWS, GCP, and Azure for this specific workload") that current retrieval systems handle poorly. This represents a meaningful architectural departure from existing vector similarity search + LLM synthesis pipelines.

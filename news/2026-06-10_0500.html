📰 本文 + 和越対訳

あるQAエンジニアが、機能変更時のコード調査時間を大幅に削減するAIスキルを自ら開発した経緯を詳細に公開した。従来のトップダウン型のコードトレースは深いネストで行き詰まることが多かったが、このスキルでは「画面に表示されるものはDBカラム・翻訳キー・フィーチャーフラグ・ユーザー権限・設定値の5種類しかない」という洞察をもとに逆引き検索を行う。対象の要素をキーとして全参照箇所を網羅的に列挙することで、AIが他機能への影響やテストカバレッジの空白を自動的に洗い出す仕組みだ。この5分類はそのままAIへの明確な指示としても機能するため、プロンプトの曖昧さを排除できる利点もある。実際の導入後は通常シナリオ・エラーケース・状態バリエーション・権限別動作を網羅したテストシナリオが自動生成されるようになったという。

Một kỹ sư QA đã chia sẻ chi tiết về cách anh tự xây dựng AI skill để rút ngắn đáng kể thời gian điều tra code khi thay đổi tính năng. Thay vì theo dõi luồng code từ trên xuống (dễ bị mắc kẹt ở nesting sâu), skill này dựa trên nhận thức rằng "tất cả những gì hiển thị trên màn hình đều thuộc 5 loại: DB column, translation key, feature flag, user permission, và config value." Từ đây, AI thực hiện tìm kiếm ngược — liệt kê toàn bộ chỗ tham chiếu đến phần tử đó, tự động phát hiện ảnh hưởng đến các tính năng khác và khoảng trống trong test coverage. Cách phân loại 5 hạng mục cũng đóng vai trò là prompt rõ ràng cho AI, loại bỏ sự mơ hồ. Sau khi áp dụng, test scenario bao phủ trường hợp bình thường, lỗi, biến thể trạng thái và hành vi theo quyền hạn được tự động sinh ra.

Claude Codeを用いたアカウント管理システムの仕様書生成において、Markdown・HTML・インタラクティブHTMLの3フォーマットを実際のトークン消費量と処理時間で比較した詳細レポートが公開された。Markdown出力の基本形は約1万3700トークン・3分で完了する一方、インタラクティブなHTML（折りたたみ・タブ・ホバーツールチップ付き）では同じ内容が11万4100トークン・33分超を要した。HTMLで生成された仕様書はSVGダイアグラムや色分けテーブルを備え、「触りたくなる」視認性を持つが、反復開発サイクルにおけるコストは現実的でないと筆者は結論づけている。実用的な選択肢として、初回の概念検証ではMarkdownを使い、最終的な共有版のみHTMLにするという段階的アプローチが提案されている。

C-03Is Grep All You Need? How Agent Harnesses Reshape Agentic Search Performance

📰 Article (English)

A new paper from researchers challenges the prevailing assumption that vector retrieval is universally superior for agentic search. The study compares grep-based (exact/pattern matching) versus vector retrieval approaches within LLM agent architectures using two experiments across Claude, Codex, and Gemini models with both custom (Chronos) and provider-native harnesses. Key finding: grep generally yields higher accuracy than vector retrieval in their experimental setup. More surprising, the agent harness itself — how tool calls are structured, results presented, and conversation managed — significantly influences overall performance independent of which retrieval method is used. Identical conversation data produced meaningfully different outcomes when run through different harnesses. Experiment 2 demonstrated a critical practical concern: adding irrelevant context progressively degrades both retrieval methods. This "noisy context" degradation pattern has direct implications for production RAG systems that pull broadly before ranking. The authors recommend practitioners evaluate retrieval strategies within their specific agent architecture rather than assuming any universal best practice. The interaction between retrieval format and model behavior is non-trivial — a finding that should inform how teams architect their agentic search pipelines.

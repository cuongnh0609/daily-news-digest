ビズリーチのAI Product Studioが発表した「Agentic UAT」は、LLMエージェントにペルソナ（人物設定）を与え、エンドユーザーとして実際にプロダクトを操作させることでUAT（ユーザー受け入れテスト）を自動化する手法だ。従来のUT・E2Eテストとは異なり、実装者の視点ではなく「業務として使えるか」を問うUATは、仕様の穴や使いにくいUIを検出する上で本質的な価値を持つ。本手法では、エージェントにAPIアクセスやソースコード参照を禁じ、UIのみで判断させることで素人ユーザーを再現する。複数のペルソナによるマルチアクター対応や業務フロー定義を受け入れ基準とする設計も特徴で、実験ではCriticalバグを含む複数の課題を自動検出できた。問題を3段階（Critical/Major/Minor）に分類し、修正ループまで自動化したことで、QAのサイクルを大幅に短縮できる可能性が示された。課題として視覚認識の限界やペルソナの行動再現性のばらつきも指摘されているが、従来のテスト自動化では捉えられなかった上流品質の担保に向けた新しいアプローチとして注目されている。

"Agentic UAT" do AI Product Studio của BizReach công bố là phương pháp tự động hóa UAT (User Acceptance Testing) bằng cách giao cho LLM agent một persona (nhân vật giả định) và cho nó thao tác thực tế với sản phẩm như một end user. Khác với UT hay E2E test thông thường, UAT tập trung vào câu hỏi "có thể sử dụng cho công việc không?" từ góc nhìn người dùng — có khả năng phát hiện lỗ hổng trong spec và UI khó dùng mà test của người implement không thể thấy. Phương pháp này cấm agent truy cập API hay đọc source code, chỉ được phán đoán qua UI để mô phỏng người dùng không chuyên. Thực nghiệm cho thấy hệ thống đã tự động phát hiện nhiều vấn đề bao gồm cả bug Critical. Việc phân loại vấn đề thành 3 cấp và tự động hóa cả vòng lặp sửa lỗi mở ra khả năng rút ngắn đáng kể chu kỳ QA. Đây là hướng tiếp cận mới đầy hứa hẹn để đảm bảo chất lượng ở giai đoạn thượng nguồn — điều mà các công cụ tự động hóa test truyền thống không thể đạt được.

B3MicrosoftのAgent Package Manager (APM) — 複数AIエージェントのハーネス管理を統一

📰 本文 + 和越対訳

Microsoftが開発した「Agent Package Manager (APM)」は、GitHub Copilot・Claude・Cursorなど複数のAIエージェント向けに指示書やスキルを統一管理するためのツールだ。複数のエージェントに同じルールを適用したい場合、従来はファイルの手作業コピーが必要だったが、APMを使うことで「apm install」一つで全ハーネスに展開できる。apm.ymlに依存関係を宣言し、apm.lock.yamlが40桁のコミットハッシュを記録して再現性を保証するのは、npmやpipと同様の設計思想だ。セキュリティ面では「File presence IS execution（ファイルが存在するだけで実行される）」モデルに基づき、apm-policy.ymlで組織ガバナンスを実現する。新メンバーのオンボーディングも「git clone && apm install」のみで完結し、CI/CDパイプラインにapm auditを組み込むことでポリシー違反パッケージを自動ブロックできる。エージェント指示書の標準化・バージョン管理が課題になっている組織にとって、実践的な解決策として注目される。

"Agent Package Manager (APM)" do Microsoft phát triển là công cụ quản lý tập trung các instruction file và skill cho nhiều AI agent như GitHub Copilot, Claude, Cursor. Trước đây, khi muốn áp dụng cùng một tập quy tắc cho nhiều agent, người dùng phải copy file thủ công, nhưng nay chỉ cần một lệnh apm install là có thể deploy lên toàn bộ harness. Triết lý thiết kế tương tự npm/pip: khai báo dependency trong apm.yml, file apm.lock.yaml ghi lại commit hash 40 ký tự để đảm bảo reproducibility. Về bảo mật, dựa trên mô hình "File presence IS execution" và dùng apm-policy.yml để quản trị tổ chức — có thể block tự động các package vi phạm policy qua CI/CD pipeline. Onboarding thành viên mới cũng chỉ cần git clone && apm install. Đây là giải pháp thực tiễn đáng chú ý cho các tổ chức đang gặp khó khăn trong việc chuẩn hóa và version control instruction file cho AI agent.

C1GitHub Copilot Code Review to Start Consuming Actions Minutes — June 1, 2026

📰 Article (English)

GitHub has announced that starting June 1, 2026, Copilot code review will begin consuming GitHub Actions minutes in addition to AI Credits. The change stems from Copilot's code review running on an "agentic tool-calling architecture" that spins up GitHub-hosted runners — the same infrastructure underlying Actions workflows. Previously, this compute cost was absorbed without separately billing Actions minutes; from June 1, that changes for private repositories. Each Copilot code review will be billed in two ways simultaneously: AI Credits (covering model inference) and Actions minutes (covering runner compute). Standard GitHub-hosted runners draw from each plan's included monthly Actions minutes first, with overages charged at the standard per-minute rate. Larger runners have their own multipliers. Public repositories are unaffected — Actions minutes remain free for them. To prepare, teams should review their current GitHub Actions usage, adjust spending limits in organization billing settings, and monitor via the Actions usage metrics API. The agentic code review feature is available on Copilot Pro, Pro+, Business, and Enterprise plans.

C2Microsoft VibeVoice: Open-Source Speech-to-Text with Speaker Diarization, Runs Locally on Apple Silicon

📰 Article (English)

Microsoft released VibeVoice, an MIT-licensed speech-to-text model with built-in speaker diarization, available January 21, 2026 and reviewed in depth by Simon Willison this week. Unlike Whisper alone, VibeVoice produces structured JSON output containing text segments, timestamps, duration, and speaker identity in a single pass — no post-processing step needed to identify who spoke when. Simon tested it on an M5 Max MacBook Pro (128GB RAM) using the mlx-audio library with the 4-bit quantized model (5.71GB). A 99-minute podcast was transcribed in approximately 8 minutes 45 seconds, peaking at 30.44GB RAM during generation. The default token limit of 8,192 handles roughly 25 minutes of audio; longer content requires bumping to 32,768 tokens. The model accepts .wav and .mp3 and has a one-hour maximum per run — split with overlap for longer content to maintain speaker ID consistency. The output quality was strong: VibeVoice reliably distinguished between hosts and guests, including sponsor-read segments. The model runs entirely locally with no API key or server required.

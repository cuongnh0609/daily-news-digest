B1DatabricksのOSSメタハーネス「Omnigent」——複数AIエージェントを一元管理する新アーキテクチャ

📰 本文 + 和越対訳

Databricksが開発したオープンソースのメタハーネス「Omnigent」は、Claude CodeやCodexなど複数の異なるAIエージェントフレームワークを統一的なインターフェースで管理・実行するプラットフォームだ。従来、複数のエージェントを利用する際はそれぞれを個別に起動し、結果を手動でコピーして連携させる必要があった。Omnigentはこの課題を解決するために、エージェントの実行を標準化する「Runner」と、履歴・ポリシー・成果物・共有機能を管理する「Server」の2層構造を採用している。ユーザーはYAMLファイルで各エージェントの役割とモデルを定義でき、複数エージェントへの同時クエリや相互批評（ディベート機能）も実装されている。組織レベルのコスト上限設定や人間の承認ワークフロー、URLによるセッション共有など、チーム開発に必要な機能が一通り揃っている点が特徴だ。マルチエージェント環境の本格普及を見据えた抽象化レイヤーとして注目されている。

"Omnigent" — meta-harness mã nguồn mở do Databricks phát triển — là platform quản lý và thực thi nhiều framework AI agent khác nhau như Claude Code hay Codex thông qua một interface thống nhất. Trước đây, khi dùng nhiều agent, người dùng phải khởi chạy từng agent riêng lẻ và copy kết quả thủ công để kết nối chúng. Omnigent giải quyết bài toán này bằng kiến trúc 2 tầng: "Runner" chuẩn hóa việc thực thi agent, và "Server" quản lý lịch sử, chính sách, artifacts và tính năng chia sẻ. Người dùng định nghĩa vai trò và model của từng agent bằng file YAML, hỗ trợ truy vấn đồng thời nhiều agent và thảo luận phản biện giữa các agent (tính năng debate). Nổi bật là các tính năng cần thiết cho phát triển nhóm: giới hạn chi phí ở cấp tổ chức, workflow phê duyệt từ con người, và chia sẻ session qua URL. Đây là lớp abstraction được chú ý như bước tiến hướng tới môi trường multi-agent thực sự trong doanh nghiệp.

C2GitHub Agent Finder GA: Copilot Now Auto-Discovers MCP Servers and Tools Using Open ARD Spec

📰 Article (English)

GitHub has launched Agent Finder for GitHub Copilot as generally available, a discovery system that allows AI agents to automatically find the right MCP servers, skills, canvases, and tools for a given task without manual configuration. Instead of developers hand-picking capabilities, Copilot can now describe a task in plain language and the system searches an indexed catalog to return ranked matches. "Agents load what the work calls for instead of carrying every tool just in case," GitHub explained, enabling just-in-time capability loading rather than permanently attached toolchains. The system implements the open Agentic Resource Discovery (ARD) specification, co-developed by GitHub, Google, GoDaddy, Hugging Face, and Microsoft. This standardization means the ARD spec can be adopted across registries and AI clients beyond just GitHub Copilot. Enterprises can point Agent Finder at GitHub's public catalog or private registries, with governance controlled through enterprise-managed settings that determine which resources agents are permitted to discover. Critically, the system requires explicit installation — "Agent Finder finds the right tool at the right time. It doesn't silently connect anything" — preserving human oversight. Available across all Copilot plans.

🔴 HIGH Foundational shift in how AI agents discover and integrate tools — the ARD open spec could become the standard across multiple AI platforms.

OpenRouter ran a novel agentic benchmark: 11 LLMs competed in a custom 2D battle royale game across 30 matches, each model controlling agents via tool calls with access to weapons, vehicles, healing items, and a shrinking zone.
